深夜日记（23）

实习的第一天

早上6:30起床，接近7点的时候洗漱完，然后整理下去楼下吃早饭（一碗白粥+两个肉包），吃完7：20，走向校门口乘公交车。今天时间凑的刚好，我离车站还有50米的时候就看到车在停站了，于是奋力冲去，司机师傅也很好心的起步之后又停下来等我（有被感动到），上车急忙礼貌地说了句“谢师傅”~

到公司之后，签了份协议，领了员工牌（卡片印着一只萌萌的猿猴<(￣︶￣)>），领了一台笔记本，介绍了主管（所谓导师，一下简称洋哥），把我的位置安排到他旁边，然后就开始工作了。

哈哈~我见到导师的时候，心里一惊，“咦！这不是那个面试我的技术官吗？”

洋哥说这几天我主要是熟悉下公司环境，不会给我太多任务，然后加了各种群。我也突然发现，前端组算上两个实习神只有5个人，按照洋哥的意思是说现在是业务驱动技术，然后前端方面也没有大量负责的需求，总体来说要求不是很高，再说公司业务也不是很多，相应的人也就少了。

早上的时候在配置我新领的笔记本，首要任务当然是先安装好chrome并把翻墙搞定！（不能翻墙的前端，不是一个好前端<(￣ˇ￣)/）然后把sublimeText3以及它的一些必备插件装起来，再装NodeJS和Git，就算是搭建好一个大的前端环境了。

中午的时候，洋哥带我们去了鱼味馆请我们吃了顿好的，途中还聊了许多。回去之后我的任务就是就开始看一个淘宝团队建的一个用于适配各种移动设备库flexible，其实原理还蛮简单的，就是：

  1. 利用meta标签中信息或者检测移动设备型号确定设备像素比dpr(device pixel ratio)以及缩放比scale，
  2. 根据设备的宽度确定相对大小rem(rem=width/10)，并为htm标签设置font-size：rem px;,
  3. 监听窗口大小变化，如果300ms内大小没有变化，则重新获取设备宽度并为htm设置font-size属性。
  4. 然后在写css的时候只需要把px换算成em就好了，结果值 = 实际px值/rem，比如设备宽度是750px的，那么rem就是75px，如果一个盒子的宽度测得是75px,那么写css的时候写成1em即可。

推荐读读这篇文章。当然我觉得源码还有有些问题的，比如

这个分支中下的if (devicePixelRatio >= 3 && (!dpr || dpr >= 3)) { }里&&后半部分的判断好像没有什么用，可以去掉。然后他对于ios下的注释也有问题。
