Node.js 学习笔记—— url querystring 模块

URL (stable)

方法

url.parse(urlString[, parseQueryString[, slashesDenoteHost]])
  • parseQueryString: 布尔值，默认为 false 。如果为true，则返回的 URL 对象的 query 属性奖总是被设为 querystring 模块的 parse() 方法返回的对象。如果为 false ， 则返回的 URL 对象的 query 属性将是一个未被解析的字符串。
  • slashesDenoteHost: 布尔值，默认为 false 。如果为 true ，则字符 // 之后， 下一个 / 之前的字符串将被解释为 host 。例如，给 //foo/bar ，返回结果将是 {host: 'foo', pathname: '/bar'} 而不是 {pathname: '//foo/bar'} 。
url.format(urlObject)
  • urlObject: 对象或字符串。 如果是一个字符串，通过把它传给 url.parse() ，转换为一个对象。

返回一个衍生自 urlObject 的格式化的 URL 字符串。相当于 url.parese() 的逆向输出。不过它的解析规则还是蛮繁杂的，具体请参考官网。

url.resolve(from, to)
  • from： 要被解析的基础 URL 。
  • to: 被解析的 HREF URL 。

用一种类似于网页浏览器解析一个锚标签的方式，相对于一个 基础 URL ，解析一个目标 URL 。

忽略字符

URL 仅被允许包含一个确定范围的字符。空格和下面这些字符将在 URL 对象的属性中被自动忽略

如，ASCII 空格 字符被编码为 %20 ，斜杆 / 被编码为 %3C 。

Query String (stable)

方法：

querystring.parese(str[, sep[, eq[, options]]])

str: 要解析的 URL 查询字符串

sep: 默认为 ‘&’ ，用于在一个查询字符串中界定键值对的子串。

eq: 默认为 '=' ，用于在一个查询字符串中界定键值的子串。

options: 对象

  • decodeURICompnent: 函数，默认为 querystring.unescape() 用于当在一个查询字符串中解码百分比编码字符时。
  • maxKeys: 数字，默认为 1000 。指定能解析的最大键数。指定 0 来移除键的计数限制。

把一个 URL 查询字符串解析成一个键值对的集合。

注意，该方法返回的对象没有原型地继承 JS 的 Object 对象，因此没有 toString() 等方法。

默认在查询字符串中的百分比编码字符被假定使用 UTF-8 编码。如果一个替代的字符编码被使用了，则 decodeURIComponent 选项需要被指定。

querystring.stringify(obj[, sep[, eq[, options]]])

obj: 把一个对象序列化为一个 URL 查询字符串。

options:

  • encodeURIComponent: 函数，默认为 querystring.escape() 用于当在一个查询字符串中将 URL-unsafe 字符转换为百分比编码时。

通过遍历对象自身的属性，生成一个 URL 查询字符串。

默认在查询字符串中的百分比编码字符被假定使用 UTF-8 编码。如果一个替代的字符编码被使用了，则 encodeURIComponent 选项需要被指定。
