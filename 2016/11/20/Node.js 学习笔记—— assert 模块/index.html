Node.js 学习笔记—— assert 模块

发表于 2016-11-20   |   分类于 Node.js   |     |   阅读次数

前言

最近在看 Node.js v6.9.1 的官方文档，发现有点儿难啃。我认为是官方文档介绍顺序的问题（当然可能是我太笨了），它没有对 API 作适当的分类和排序，只是按照字母顺序粗暴地罗列了API，这导致文档条理不清晰，看起来比较费劲。因此，我对几个我认为比较重要的模块进行再整理分类，下面是一些笔记。

Assertion Testing模块（Locked）

  • 可用于测试常量；
  • 旨在用于内部使用，但是可通过require('assert') 被用在应用中；
  • 不是测试框架；

抛出错误

assert.fail(actual, expected, message, operator)

  • 抛出 AssertionError

  • 如果 Boolean(message) === false，则抛出 AssertionError: ${actual} ${operator} ${expected} 。

  • 如果 message 被定义了,则抛出 AssertionError: ${message}

    ​

判断一个值是否为true

assert.ok(value[, message])
  • 如果 Boolean(value) === true，则ok
  • 否则，抛出 AssertionError: value == true 或 AssertionError:${message}
  • 也可以用 assert(value[, message]) 表示
assert.ifError(value)
  • 如果 Boolean(value) === true，则抛出 Throw ${value}
  • 否则为 ok

判断两个数是否相等

assert.equal(actual, expected[, message])

  • actual == expected

assert.strictEqual(actual, expected[, message])
  • actual === expected
assert.deepEqual(actual, expected[, message])
  • actual == expected

  • 只有可枚举的自身属性会被考虑。（不测试对象原型，附加 symbols ，或者不可枚举的属性），只要两个对象的属性一一对应，且值相等，则两者相等。

  • 深度相等意味着，子对象的可枚举的自身属性也会被测试

assert.deepStrictEqual(actul, expected[, message])

  • actual === expected

  • 会比较对象的原型

assert.notEqual(actual, expected[, message])
assert.notStrictEqual(…)
assert.notDeepEqual(…)
assert.notDeepStrictEqual(…)

判断一个函数是否有抛出指定错误

assert.throws(block[, error][, message])

  • 立即调用block函数

    • 如果没有抛出错误，则抛出 Assertion: Missing expected exception (${error}) .$(message)
    • 如果抛出了错误，且类型和 error 匹配，则ok
    • 如果抛出了错误，但类型和 error 不一样，则将会继续抛出原来的错误。

  • error可以是构造函数、正则表达式、验证函数（但是不能为字符串）

assert.doesNotThrow(block[, error][ , message])

  • 会立即调用block函数

    • 如果抛出了错误，且类型和error一样，则抛出错误提示Throws: AssertionError: Got unwanted exception (${error}. ${message})
    • 如果抛出了错误，但类型和 error 不一样，或者 error 没有指定，则会将错误传回调用方，即抛出原来的错误。
