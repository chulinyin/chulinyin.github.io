沉默的大多数

发表于 2016-09-22   |   分类于 读书随感   |     |   阅读次数

开始第二遍看《思维的乐趣》了，我会对我觉得有意思的部分做摘录，然后讨论一番，希望能够帮助我在思维中享受到某些乐趣。

人要永远做小孩子虽办不到，但想要保持沉默是能办到的。在公众场合什么都不说，到了私下里则妙语连珠，换言之，对信得过的人什么都说，对信不过的人什么都不说。（中国人的通病？）

所以这里的保持沉默是在相对情况而不是绝对情况下的吗？对信不过的人保持沉默，对信得过的人，畅所欲言。那如何判断一个人是否信得过？

我所理解的“信得过”是指，你相信他不会把你的言论传播出去，即使传播也必定有所保留，不会把你暴露，总而言之就是不会让你受损，甚至会让你受益。然而受损和受益的判断标准又是相对的，每个人的标准有所不同。比如说，他可能认为使你头发短些是无损的，甚至是有好处的，可是你可能认为这样会让你受损。所以，信得过也是相对的？

然而按作者所说，他把人泾渭分明地分成两类，然后对之以完全相反的两种态度，真的能够做得到吗？判断信得过的依据又是什么呢？

我自问做不到如此，而且我认为作者想要说明的可能是“沉默和信得过应该成反比，也就是对越信不过的人，越保持沉默”。可是这样一来情况变得更加复杂了，因此我这个认为极有可能是错误的。所以，我现在困惑的是，分界线到底在哪里？纯粹依靠个人直觉吗？

深夜日记-关于收获

发表于 2016-09-13   |   分类于 日记   |     |   阅读次数

又有好久没有写点东西了，之所以拖着不写是因为相比之下我有更加重要的事情需要完成（当然也不排除是自己太懒的可能）。

快速地学习前端工程化的知识（vue+webpack），并尽快产出一个项目。

刚开始时因为不知道究竟去做什么项目，于是就决定先简单做一个自己的网页版简历试试水。但途中却意外地涌现出很多奇妙的想法，把这些想法拼凑起来就是一个大型项目的雏形。这点让我很是兴奋~因为首先这个项目是有实际意义的，甚至是可以产生盈利的，其次如果能够顺利完成它，我想我就能突破现在的瓶颈，顺利进入到下一个阶段了。但是恐怕得等到校招之后我才会去动手实现它，在此之前还是先尽量把我那个简单的网页版简历做得尽善尽美，然后搞懂vue和webpack。

把自己思想中一些不好的东西丢掉，重新形成自己的思想内核雏形。

这么说可能有点玄乎，我可以详细解释一下。

大概在一个月前，我开始思考我作为一个独立个体应该如何去提高自己。

这样说可能太过简单，提高自己所指也有些虚无缥缈。于是我当时开始想，除去外物后，人所拥有的一些东西是什么？我总结后发现可能是下面这些：

  • 外在：脸、身高、穿着、身材、品味……
  • 品质：明辨是非、诚实守信、负责……
  • 能力：听说看写，以及最重要的思想内核。

所以我就准备从这三方面开始着手提高自己。

外在

外在除了身高，其他其实都可以通过自己的行动去改变。虽然外在相对于另外两种可能一文不值，但是它还是有很多好处的。你的外在是别人对你第一印象的基础，而让他人有个好印象还是挺好的，所以提高外在似乎是一个不错的选择。另外一个健康的身体是一切的基础，没了它一切都是白搭。所以我决定开始定期锻炼、学习基本的穿衣搭配、尽力提高自己的审美品味。

品质

品质这种东西看不见摸不着，但是却能够被感知。培养一些优秀的品质，也是提高自己的一种方式，但是哪些才是优秀的品质呢？爱国？孝顺？不不不，自从看了王小波的《思维的乐趣》，我认为明辨是非才是最重要的品质，同时也是最难培养的品质。当自己越来越能够明辨是非的时候，其实也就知道哪些品质是优秀的了~

那明辨是非的定义是什么呢？简单来说就是能够分清一件事情的因果、好坏吧~那怎么培养呢？多用正确的方式思考！比如：把个人情感放一边，基于证据去判断一件事情，在证据不充分的时候不下定论，当发现证据错误的时候及时修正判断 ；观察地更精确一点、估量事情更仔细认真一点、做决定更小心一点。

能力

最后就是能力了，本来我以为能力应该是指自己现有的技能，但是我突然发现这样想太过于狭隘了，相比之下更重要的是理解事物、学习事物和输出有价值的东西的能力。综合概括就是听说看写。也就是人的输入和输出功能，

当然最重要的还是大脑处理输入的能力，也就是最重要的思想内核。所以我先把输入和输出能力放在一边，开始想办法提高自己的思想内核。可是怎么才能提高自己的CPU呢？一个种可能是去不断处理输入，渐渐形成自己的思维方式，但是这样有点慢而且可能会把自己搞糊涂。于是我就想，只能先直接吸收优秀的人的思想了，就是说直接先假定那种思想是好的。这么做可能很快，但是风险极大！所以挑选优秀的人写的优秀的书成了至关重要的一步！

其实有一种很有意思的现象，如果你看了一本好书，你可能就会在这本书中发现其他好书，然后一路发现好书。但是如果你看了一本坏书，那自然可能就是一路坏书了。这种现象类似于自动控制系统中的正反馈，好的越好、坏的越坏。

那怎么找好书呢？《论语》、《大学》……这些中华古典名著算不算是好书呢？因为我没有通读过，所以我不知道，但是我想我是不会再去读它们了，除非哪一天我需要一些反面教材。我先来说说我看过的并且自认为是好书的书：《思维的乐趣》、《黑客与画家》、《Beyond feelings: a guide to critical thinking》、《鲁迅全集》，这些是能够直接产生作用的书，而且这些书里面也有一些优秀的人以及一些好书。还有一些书可能是潜移默化的，其中道理是需要自己去领悟的，通常是小说：《兄弟》、《活着》、《许三观卖血记》、《三体》、《红与黑》，这些是我看过小说中自认为比较好的。

发现和承认自己的不足，并有改进的欲望。

这可能是最为重要的收获了，一直以来我都没有刻意地去发现发现自己的不足，也没有刻意地想着去提高。可是现在因为要开始面对自己将步入社会并完全独立这个事实，我也终于开始去思考未来的人生，开始思考以前完全没有想过的东西。

昨天刚看完王小波的《思维的乐趣》，收获可以用巨大来形容吧。这个暑假干的最明智的事情可能就是看了这本书和重看了《黑客与画家》。遇到一本好书，真的是时看时新啊~说实话，真的很羡慕他们的智慧、能够透过现象看到本质的一些东西的能力、能够较好地控制自己的情绪的能力、能够早早就心智成熟，知道自己真正需要什么。

我才21，未来的路还很长。

我已21，人生的三分之一。

一封离职信

发表于 2016-09-03   |   分类于 工作   |     |   阅读次数

公司氛围挺好的，
同事也都挺友善的，
工资的话，作为实习生也能接受。

但是作为一名技术人员来说，其实比较看重的还是公司业务对自己技能和项目经验的提升。而且这两者也是相辅相成的：技术人员技能的提升必然使公司的业务需求能够更好更快地完成，而公司进一步增长的业务需求也必然使技术人员不断学习，不断提升自己的技能。

然而在实习的俩个多月里，公司对我技能提升的帮助，除了一开始的两个礼拜就几乎再也没有了。（当然，你可能认为难道就不能自己学习提升技能了吗？首先，我当然有在空闲的时间去学习新技术，去提升自己的技能。可是在外部压力下学习的状态和在自我驱动下学习的状态还是有很大的区别的，人在外部压力下，往往能够更加快速地学习和应用技术。这也是我不在家里学习，而是来公司实习的原因。）其实本来我还是抱有一线希望的，因为主管洋哥提议要前后端分离，可是遗憾的是最终老板没有同意。所以我很害怕如果我再继续呆下去，我的技能就不进反退了，而这种恐惧恰恰是一个技术人员最害怕的。

另外淘粉吧前端分配到我这边几乎没有什么有挑战性的活。其次，我也没有感受到公司对新人培养的用心，感觉对实习生并不重视，当然这些我也都能试着理解~

以上就是我想说的。最后感谢公司对我这段时间的培养，让我从中学得到了许许多多的东西。
也祝愿我们日后各自安好~

反思

我以为我可以成功，我以为我只差那一步了，现实却告诉我，其实我还差得很远很远。心突然之间就静了下来，开始对自己进行反思。

  • 不得不承认，时间不够确实是一个很有用的借口，用来骗自己慢慢来不要着急，再适合不过。
  • 不得不承认，我有点儿井底之蛙，自以为是了。自以为很牛逼，其实也就那样。
  • 不得不承认，心有点儿浮躁了，没有专心地去学习本该学的东西，还把过错归咎到其他地方。

这当头棒喝，这一盆凉水，也终于让我的躁动再次归于平静。也让渐渐放松警惕的我，渐渐开始自满的我，重新回到应该有的状态。

早起日记-我的前端之路

发表于 2016-08-31   |   分类于 工作   |     |   阅读次数

突然发现又有好长时间没有写点东西了，虽然还是有许多东西值得一写的，但是可能是因为自己又变懒了，可能是因为突然又失去了持续写博客的动力，也可能是因为等网易的内推结果等得心力憔悴没有心情写……总之，推脱的借口总是可以找出一箩筐。

今天索性就把自己一些零碎的想法写下来吧……

关于前端

距离我报名网易云课堂的前端微专业，真正走上前端这条路，已经过去8个月多了。时间不长也不短，简单回顾一下自己的经历，希望能对我有一些帮助~

花了4个月左右的时间顺利完成了第一阶段目标：达到了微专业的优秀学员标准，一只脚成功跨入了前端大门。

一开始的我除了会点PS什么都不懂，于是一边看课程视频一边做笔记。当我做完两本笔记的时候，课程也快结束了，接着开始挑战最后的大作业。当时的我痛并快乐着，因为我能明显地感觉到自己的进步，同时又苦于自己学的时间太短，很多地方都是一知半解，做得比较辛苦。不过，所幸赶在deadline之前完成了。虽然代码有点儿粗糙，效果也不是十分完美，但是完成后带来的那种成就感真的无法言语~（而这可能是我永远无法从单片机编程中获得的东西）

花了1个月左右的时间顺利完成了第二阶段目标：补足基础，另一只脚也跨入前端大门，找到实习。

然后为了补足自己的基础，就开始啃高程（那时候经常能在图书馆看得睡着），一天下来只能啃20多页（而且这还是在没有做笔记的情况下）。由于学校里还有其他事情，于是期间不停抽出时间来看书，好在那学期的课不多，在5月13号终于把高程啃完了。看完高程的我终于能看懂一些简单的组件，终于了解前端的大致模样了。

然后为了让自己能够顺利进入第三阶段，我马上就开始找实习了（其实本来是打算3、4月份就开始找的，但当时对自己实在是一点儿信心都没有，也就没敢去找。）。投了几家公司，最后面试了两家公司，6.21开始进入第二家公司实习。

第三阶段：达到自己想去公司的要求。

我已经在这家公司实习两个月多了，那么我进入到自己想要的第三阶段了吗？我自己认为还远远没有。我当时给自己的暑假实习计划大概是这样的：

  • 深入一个项目，学以致用，不停掉坑不停爬坑，增加自己的项目实战经验
  • 学习工作中的前端开发模式，了解并运用工程化的前端工作流
  • 能够熟练使用一种框架（vue或angular或react）
  • 学点数据结构和算法
  • 能够在好师傅的带领下走向阳光大道

然而现实真的太残酷了：

  • 公司交给我的任务只是几个简单的活动页，而且真的很闲。
  • 公司前后端没有分离，还处在刀耕火种的年代，近期主管想要前后端分离却被保守的BOSS否决了。
  • 公司没有使用框架，大多数都是用JQ解决。
  • 算法书实在看不下去
  • 师傅？唉……算了，还是只能靠自己啊~

虽然我尽量利用自己空闲的时间去学习我所计划的东西，也有许多的收获，但是速度真的太慢太慢了，我想要的是能够在秋招的时候达到自己想去公司的要求啊~

当然最大的原因还是在于我自己吧~自我控制力还是不够强，内心也不足够强大，智商也一般，努力程度也一般，远远没有到拼命的程度，真的狠不下心对自己太狠。

那有什么办法能够跳出这个困境呢？我想了一下，只有两条出路了：

  1. 对自己再狠一点，压迫自己快速进入到第三阶段。
  2. 换一家公司，遇到一个好师傅。

虽然换一家公司应该能够做到，但是公司的环境能够符合甚至超出我的要求的几率好像有点儿少。那么，只能对自己狠一点的意思咯？

故事还很长，未完待续……

我的实习之路（四）

发表于 2016-08-22   |   分类于 工作   |     |   阅读次数

融入团队

上一篇说到作为一个职场新人（特别是刚从象牙塔里出来的大学生），因为几乎没有什么社会经验，难免会遇到许多困惑。我想最大的不适应可能是突然从「与某个人交往」变成了「与一群人交往」，所以究竟如何才能「以正确的姿势」融入到一个团队中去呢？

作为一个已经快实习两个月，但却依然觉得没有完全融入到团队中的人，我仅有一些零碎又可能可笑的想法，希望有一点儿参考价值。

团队的价值观和文化

可以先体会一下所在团队或者公司的价值观和文化。如果发现团队的三观与你或者你所期望的大相径庭，甚至完全相反，那就没什么好说的了，趁早走人~而且也只有在「团队的氛围能够符合你的预期」这个大前提下，你才有动力让自己融入到团队中去。

那么“价值观”、“文化”、“三观”等这些名词指的到底是什么呢？（具体意思可以维基百科）我认为是一个行为处事的标准，当为一道没有答案的选择题做出一个选择时，往往就是这些内在的东西在起着巨大的作用。

我个人比较向往的几个公司的内核：谷歌的“不作恶”、锤子科技的“天生骄傲”、网易的“有态度”。

有时候其实你是对的，但是因为周围错的人太多，你因此产生了自我怀疑，甚至最终为了迎合环境，变成了错的人。（怎么判断自己对错与否呢？多看些好书就知道了O(∩_∩)O~~）

团队负责人的态度

其次应该关注的就是团队负责人（或者说你的“导师”）的态度了。如果你很不幸地碰到了一个三观不正的“导师”，那就没什么好说的了，第二天就走人~如果整个公司，以及你所在团队，以及你的“导师”的三观都在你的接受范围之内，那你就要开始尝试自己做一些调整去融入这个团队了。

当然在这个过程中其实你的“导师”起着很重要的作用。如果你的导师虽然品行挺好，但是对你不作为，不主动帮助你去融入团队的话，这个过程还是会有点儿艰辛的，特别是对于那些不善于交际的人来说（譬如说我(⊙﹏⊙)）。如果你人品爆表，碰到一位良师，其实你要做的就是积极主动配合他（俗称“抱大腿”）就好了~

我结合自己的经历总结了几点”好导师“的品质，望我日后自己有一天能够带团队的时候能够谨记，而不是像鲁迅先生说的健忘的人一般，忘了自己曾经吃过的苦，甚至还把这种苦让别人再尝受一遍。（这当然不是原话,但却是原意，变成了自己曾经所厌恶的人，本质可能是因为健忘。）

  1. 自己做团队建设的时候，首先解决称呼问题，这样不至于有太多的官僚气息也不会让新人困惑。
  2. 尽量解答新人各方面的问题，能让他畅所欲言（当然是先经过他自己观察思考的，而不是无脑问题）。
  3. 如果新人第一次聚会，一定不要让他尴尬得无所适从，主动介绍他，让他快速融入团队。
  4. 解决新人的培养模式问题，先探测底细，然后合理安排任务和进度，而不是不管他是谁，不管他的能力如何，都特么的从头来一遍，浪费大家的时间。

作为新人需要尝试的事情

以前是觉得谁看的顺眼就慢慢成为好朋友，觉得谁看不顺眼不搭理就是了，可是这一套在社会中可能行不通。

突然想起《黑客与画家》中有这样一段话：

许多人20岁出头时感到非常困惑和压抑。大学生活很有趣，可是已经过去了。上班的日子为什么会差别这么大？不要搞糊涂了，你现在已经从顾客变成了仆人，在这种新生活中获得乐趣是可能的。不过你首先需要入门，门口的牌子写着“闲人勿进”。这种转变是一种冲击，如果你不赶快意识到这一点，事情将变得更糟。

其实与人相处的方式，在学校里与在工作中并没有什么不同，区别就是你需要学着与一群人相处，而不是我行我素地挑几个自己喜欢的人相处。善良、勤奋、真诚、守信、大度等这些品质在大多数情况下都能够获得别人对你的好感，那么如何在好感的基础之上进一步培养感情呢？（怎么听着像是谈恋爱……）

  • 稍微变得积极一些。你可能做不到主动去搭讪，但是当别人来搭理你，尝试接近你的时候，你也别高冷（其实是脸皮薄，不好意思）啊~
  • 再变得积极一些。有活动的话，尽量还是参加吧~群里也别潜太深，尽量时不时冒个泡吧~
  • 再再变得积极一些。可以自己买一些好吃的（不需要贵的），与大家分享嘛~有什么段子，也可以与大家分享嘛~
  • 再再再变得积极一些。看到别人有困难或者为难之处，主动提供下帮助~（雪中送炭才是至高境界呐~）

我们把由父母供给收入的模式称为“老爹模式”。这个模式与真实世界的最大区别之一就是对勤奋工作的评价不同。在老爹模式中，勤奋工作本事就是值得的，老爹会感到高兴。但是在现实中，财富是用工作成果衡量的，而不是用它花费的成本衡量的。如果我用牙刷油漆房屋，屋主也不会付给我额外工资的。

我的实习之路（三）

发表于 2016-08-20   |   分类于 工作   |     |   阅读次数

新人的困惑

刚进入一个新的环境中，难免有许多或大或小的困惑。有时候为了能够更好地适应新环境，融入到新环境中，我们可能需要作出一些妥协。下面是我在实习时遇到的一些困惑，小到一个称呼，大到自我发展。希望这些浅薄的思考能够有一些参考价值，也希望自己能吃一堑长一智。

称呼问题

有时候人们为了显得亲切一些，便给其他人取一些自以为好听的绰号，却丝毫不在乎本人的想法。下面可能是最常听到的一些绰号（假设全名为ABC）：阿A，阿B，阿C,小A，小B，小C，BB，CC，A哥/姐/爷/老板/总，B哥/姐/爷/老板/总，C哥/姐/爷/老板/总，……我在公司甚至第一次听叫别人“小A大师”，“XX大师”（我滴妈，我尴尬症都犯了，这年头大师变得这么不值钱了？）

所以如果你恰好有一个可能念起来不太顺口的名字，同时不喜欢被人这样随意取绰号，但是又不想显得自己高冷，那么该如何是好呢？

当然是自己想一个咯！如果你能有一个恰当的英文名，就不至于被这些诡异又变扭甚至带有阶级色彩的外号所困扰了。如果实在想不出一个好的英文名的话，与其让别人给你取阿猫阿狗的绰号，倒不如自己想一个符合自己品味的外号。

( ^_^ ) 最后记得进公司之后，主动报上自己的外号~

大家好，我叫ABC，你们可以叫我XXX哦~

哦！对了，如果你不想那样叫别人呢？先看他的绰号是不是自己给起的，以及他自己对那个绰号是否满意，以及你是否能够叫得出口，如果不是，那就直接叫名字吧！不要怕，我就是这样的~

是否要随大流

刚进入公司的时候，其实我有想自己买个显示屏，然后用双屏的。但是看到身边的人都没有用，自己用了怕让人觉得自己一个实习生太装了，最后也就不了了之了。（主要是因为我不了解公司的价值观和文化）

其实有很多地方会产生这种矛盾，你会发现你想干的事情，或者正在干的事情，有时候会与周围环境的人格格不入。虽然大多数情况下，他们不会在意你，极有可能是自己想太多。但是有些情况，有些禁区，我们确实不太敢去触碰，特别是在刚进入一个新的环境，不知道周围人的品性的情况下。

如果是你自己熟悉的环境，或者说你提前知道了周围人的品性，那当然无所谓。因为你能明确的知道，哪些行动会越线。不过这个在技术团队里其实还好，因为技术人员不会过分去关心其他的东西。

幸运的是在互联网公司，大多数的情况下你都可以继续保持自己行为处事的方式，不用刻意压抑或者刻意改变。

感觉自己不受重视

觉得自己是匹千里马，但是却没有遇到伯乐，这可能是新人会有的想法。我当时觉得，以自己的能力完全可以做一些更加有挑战性的东西。但是现实是他们不敢或者不会让新人去尝试一些他们认为有难度的事情，他们只会把一些简单的，认为新人可以完全可以胜任的工作交给他。至于为什么会这样呢？

  • 可能是新人自己没有充分地展现出来自己拥有那个能力；
  • 可能是他们怕麻烦，因为新人在挑战有难度的东西的时候，势必会遇到许多不懂的问题，特别是当涉及到公司业务的难题时，无法通过自己搜索解决。于是只能去请教他们，可是他们可能觉得这是在浪费他们时间。
  • 可能是他们害怕新人成长过快，怕积极培养新人之后，新人会超过自己。（弱者常常会有这种顾虑）
  • 可能是他们完全没有意识到人才培养的重要性，而这在技术团队里是极有可能发生的。

那么碰到了觉得自己不受重视的情况下，应该如何是好呢？

  • 积极地把简单的任务做到完美。因为即使是一些简单的任务，由于新人的经验不足，还是会遇到许多坑的。
  • 可以试着自己去争取一些有难度的任务，但是以我自己的性格来说，我是既不能也不会的。
  • 如果自己觉得工作太轻松了，那空闲的那段时间不正好可以用来提升自己吗？这时候你没有为公司创造实际的价值，但是却能拿到工资，这种情况难道不是挺理想的吗？（虽然不是你自己不想为公司创造价值）

团队架构以及自己的职责

新人可能完全不知道自己在公司里起着什么样的作用。

  • 一种是可以直接自己去问，一般公司都会有一张清晰的组织架构图，以及职位所对应的职责。
  • 可是像我就居然特么的拉不下脸去问（我在陌生环境下，脸皮是有多薄哦？），这时候就需要自己去观察，感受整个公司的运作流程。

新人也可能不太清楚自己的职责到底是什么。

  • 可以去看招聘需求，你既然能被公司招进来，想必大部分原因是因为你能够满足对应职位的要求。
  • 可以直接问你的主管，他也会明确告诉你所在部门负责干什么，具体分配到你这边需要完成什么。

这一系列连载不知不觉居然已经拖到第二个月了(掩面狂奔)……

  • 算是成功拿起了前端大门的敲门砖。
  • 对自己以后想去什么样的公司和团队有了更具体的认识。

当然同时对自己、对公司也有一些不满意的地方：

  • 对自己不够狠，虽然总体上还是每天都在进步，但是速度真的太慢了。
  • 公司的氛围确实很不错，同事也都很友好。但是给我的工作量实在太少太少了，闲得慌。（当然其实不是给我的工作量太少了，而是总体的工作量就不大）
  • 我没有感到公司（或者说自己的主管或者说自己的“师傅”），有在花心思培养我，帮助我快速成长，真的一点儿都没有感受到，这大概是最失望的一点了。（不过祸福相依，让我更加坚定了靠人不如靠己的想法，也让我没有了牵挂）

以上算是对这两个月实习以来的一次大总结了~之所以总结得如此粗糙，没有详细深入进去，是因为觉得写那么细意义不大，自己知道就行了。

渲染树构建、布局及绘制

发表于 2016-08-14   |   分类于 CSS   |     |   阅读次数

自认为运气比较好地过了线上笔试，过两天就要去面试了，于是开始找一些前端面试题作为准备。由于前端涉及的领域实在是太广，整理的过程挺艰辛的，不过也收获了很多以前没有了解到的知识点。我会选一些比较好的放到博客上。

网页生成过程：

  1. HTML代码转化成DOM
  2. CSS代码转化成CSSOM（CSS Object Model）
  3. DOM 树与 CSSOM 树融合成渲染树（渲染树只包括渲染页面需要的节点）
  4. 布局（layout），计算渲染树每个节点的几何信息（精确位置及尺寸）
  5. 将布局绘制（paint）在屏幕上（把渲染树中的每个节点转换为屏幕上的实际像素）

渲染树的构建：

  1. 从 DOM 树的根节点开始，遍历每个可见的节点。
    • 某些节点完全不可见（例如 script 标签、meta 标签等），因为它们不会在渲染结果中反映，所以会被忽略。
    • 某些节点通过 CSS 隐藏，因此在渲染树中也会被忽略。比如有节点有一条显式规则设置了 display:none 属性，所以不会出现在渲染树中。
  2. 给每个可见节点找到相应匹配的 CSSOM 规则，并应用这些规则。
  3. 最终输出渲染树，其中包括了内容以及屏幕所有可见内容的样式信息。

CSS重绘和重排

  • 重绘：是一个元素外观的改变所触发的浏览器行为，例如改变visibility、背景色等属性。浏览器会根据元素的新属性重新绘制元素，使元素呈现新的外观。重绘不会带来新的布局，重绘不一定会引起重排。
  • 重排：渲染树重新计算。重排一定会带来重绘。一个元素的重排通常带来一系列的反应，甚至会触发整个文档的重绘和重排，性能代价是高昂的。，下面的情况会发生重排：
    • DOM元素的几何属性的变化（尺寸、位置等）
    • DOM树的结构变化（节点的增减、移动）
    • 获取某些属性（offsetTop、offsetLeft、 offsetWidth、offsetHeight、scrollTop、scrollLeft、scrollWidth、scrollHeight、 clientTop、clientLeft、clientWidth）。
    • 调整浏览器窗口的大小等
