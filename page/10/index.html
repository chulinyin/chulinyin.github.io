在向上查找过程中.parent()、.parents()、parentsUntil()、.closest()的区别：

  1. .parent([selector])
    返回节点的直接父元素组成的集合。也可以传入一个字符串选择器，用于筛选返回的元素。（因为jq可以同时选择多个对象，所以.parent()可能返回多个元素的父元素组成的集合）

  3. parentsUntil([selector ] [, filter ] )
    查找当前元素的所有的祖先元素，直到遇到选择器对象匹配的元素为止，但不包括选择器元素，filter可以用于筛选。
  4. closest([selector])
    从元素本身开始向上查找，返回最先匹配的祖先元素。(只有一个祖先元素)

.html(),.text(),.val()三个方法功能上的对比
1、.html(),.text(),.val()三种方法都是用来读取选定元素的内容。
2、.html()是用来读取元素的HTML内容（包括其Html标签）
3、.text()用来读取元素的纯文本内容，包括其后代元素
4、.val()是用来读取表单元素的”value”值。
5、和.text()方法不能使用在表单元素上,而.val()只能使用在表单元素上；另外.html()方法使用在多个元素上时，只读取第一个元素；.val()方法和.html()相同，如果其应用在多个元素上时，只能读取第一个表单元素的”value”值，但是.text()和他们不一样，如果.text()应用在多个元素上时，将会读取所有选中元素的文本内容。
javascript 中 innerHTML()、innerTEXT()、outerHTML()、outerTEXT() 的区别：
1、innerHTML 设置或获取位于对象起始和结束标签内的 HTML
2、outerHTML 设置或获取对象及其内容的 HTML 形式
3、innerText 设置或获取位于对象起始和结束标签内的文本
4、outerText 设置(包括标签)或获取(不包括标签)对象的文本

逻辑就是研究说话的‘道理’的学科

用数学集合的概念去论证一些东西

  1. 所有人都会死，苏格拉底是人，所以苏格拉底会死。
  2. 所有人都会死，苏格拉底会死，所以苏格拉底是人。

显然命题1逻辑是正确的，而命题2逻辑是错误的。可以将以上两个论证抽象为：

  1. 所有A都是B，P是A，所以P是B；
  2. 所有A都是B，P是B，所以P是A；

比较论证最好不要有
一般我们都讨厌拿别人的比较，然后要求自己像别人一样。比如：

XX家孩子学习如何用功，成绩如何好，获了什么什么奖，毕业找到了什么什么工作。你也要用功一点，这样以后能够找到好工作。

所以我们也最好不要用比较论证的方式去论证一些东西。

再读《活着》

昨天去台州找高中的好朋友，仔细算来也有一年多没有见面了，一起玩得很嗨，很开心。他是我一直想成为的类型，幽默、风趣、情商高，好像没有什么羁绊似的。可是我一直都不敢放开自己，可能是一些自卑感和莫名的责任感在作祟吧~

今天回来等火车的时候突然想再看一遍《活着》，捧着kindle就开始看了起来，等到快要下火车的时候刚好看完。

上一次看《活着》是大一暑假的时候了，那时候在余姚的书城里，坐在地砖上用一下午把这本书看了一遍。现在记忆中只记得当时很是震撼，中途眼泪有数次不自禁地流出，无声地哽咽。从此我便喜欢上了余华这位作家，然后把他写的小说《许三观卖血记》、《兄弟》、《在细雨中呼唤》都看了一遍，于是愈加喜欢这位作家。

第二次看这本书，依旧震撼，依旧不自禁地在火车上无声地抽泣和哽咽。主人公徐富贵对“我”述说了自己的一生，那是困苦和不幸的一生。还有什么比亲手害得家道中落，亲手埋葬自己的父亲、儿子、女儿、爱人、女婿、孙子更让人悲痛欲绝和不幸的人生？至少现在的我肯定无法接受这一切的折磨。

里面最让我感动的是他的爱人“家珍”和他那因高烧而不能说话的女儿“凤霞”。我不知道为什么那个时代的女性骨子里会有一股淳朴憨厚的气质，令人不得不动容。家珍即使知道富贵在外面风流也不去和他吵闹，而是通过隐晦地方式告诉他自己的想法。

家珍有一次给富贵做了四碗看上去不一样的蔬菜，富贵发现虽然四碗蔬菜都不一样，可是吃到最后都只剩一块肉在碗底。富贵知道她是想告诉他，虽然每个女人看上去都不一样，可是本质都是一样的。（女人的智慧）

在富贵把家产输光之后家珍被迫被父亲领回了家，可是她在生完儿子之后不到半年就又自己回到了富贵的破茅房，和他一起过日子。在富贵消失的两年里，始终不离不弃地照顾着富贵的家；富贵说让她去休息一会，她就感动幸福不已；在发现自己生病后，第一句问富贵的话居然是“我以后不能自己养活自己了，怎么办？”；她始终在意的都是富贵和孩子，他们仿佛就是她的一切。（写的时候。不知不觉，眼框又湿润了…..）

而“凤霞”，虽然因为高烧不能说话，但是她几乎拥有了女性的完美性格：“懂事、淳朴、勤劳、踏实、天真、纯洁”。命运给了她一个几乎是完美的丈夫，可最后又仿佛是命运的安排，在生完孩子之后就失血过多死了。

当人经历过太多的不幸，就会开始怀疑命运，怀疑自己存在的意义，不是自杀就是麻木地生存下去。可是富贵在大起大落，然后直接跌落谷底后，依然敏感地活着，他清楚地知道自己的一生，坦然地面对过去、现在和未来。这一点是最让我佩服的地方。

关于《变形记》

前天在看完《海边的卡夫卡》后就在寻思着下一本书是什么了，想了没多久就顺理成章地决定去拜读一下卡夫卡的小说，说到卡夫卡，我只能想到《变形记》和《城堡》，由于还不知道怎么在kindle上找原版的英文书，于是就先下了一个中文版的看。

阅读进度还是很快的，昨天早上看了一小时，今天早上看了一小时，就看完了。整本书有一个很大的矛盾点：

主人公对家庭的极度负责，以致于几乎没有为自己而活过，即使在变成跳蚤之后也在为家庭而担心忧虑。然而亲人在发现他变成一只无用的跳蚤后，逐渐反转以前对他的态度，变得越来越冷漠，越来越厌烦。最后主人公悲哀地饿死在自己的房间里。

除了主人公，整本小说中出现的人物冷血残酷，几乎都没有人情味可言。公司的领导只会尽情地压榨他，仅仅只是发现他一早上没有来，就直接派出顾问到家里调查情况，一发现情况不乐观就直接开除；而亲人呢？一开始妹妹和母亲还维护他（可能是抱着某种他能恢复的期望吧），可是坚持了没多久就开始对他各种厌烦和嫌弃。父亲呢？大概从一开始就想着抛弃他吧。

而最让我感到悲哀和残酷的一点就是，主人公虽然不能说话，可是他能听懂亲人的对话，能够清晰地观察和思考。这简直就是一个最大的悲剧啊！

如果我为别人付出很多很多，甚至放弃自己的个人意愿和灵魂。当某一天我突然不能够再继续为他们付出了,他们就开始抛弃甚至于欺凌我，而我只能眼睁睁地被迫接受这一切，无法反抗一丝，那时我会多绝望、痛苦和悲哀？我甚至连辱骂和诅咒的想法都懒得产生。我大概会尽早离开这个残酷的世界吧~

可是主人公还有更大的悲哀，他在这时候却没有对亲人产生巨大的怨念，他居然还为亲人的生活担忧。当一个人被别的东西俘虏，甚至于自己的整个世界都必须为它所服务的时候，他已经不能再称之为一个独立的个体的了吧？

我不知道什么时候有了这个想法：我帮助别人的时候，必须是我自愿的。而如果一旦是我自愿的，我就最好不要抱有别人会给我什么回报的想法。简单的说就是：我帮助你是自愿的，我也不会想着以后你会给我什么好的回报，甚至我还做好了你给我恶报的准备（当然如果你回报了我，我也不会矫情拒绝）。如果有了这种思想准备，当被自己曾经帮助过的人落井下石了的时候也会冷静许多吧~

关于作业

大学里有一个怪异的现象，一部分人总是等着别人完成作业，完成老师布置的任务，然后坐享其成。我也会有这种行为，虽然我觉得这是极其诡异和不正常的。这是一种怎么样的心理呢？

  • 这事对我来说是浪费时间，可是我又不得不把它完成。
  • 反正最后复习的时候可以补回来的，先抄着也是一样的。

关于实习

虽然从三月份就开始想着要找实习了，可是一直拖到现在我也还没有投出去过一份简历。原因无大概就是

  • 实力不够，以至于严重信心不足
  • 起步确实太晚了，好多需要的东西都还没了解到
  • 自己确实不足够勤奋努力，天赋也不强

关于效率

为什么自己做事效率有点低呢？感觉自己的专注程度和环境有很大关系，如果环境不好，我就很容易分心。重点还是自我控制力不够吧~

关于js的收获

今天在做百度前端学院的一道题的时候，需要验证输入字符串

  1. 城市名必须是中英文字符
  2. 空气指数必须是正整数
    第一反应就是想有没有对应的判断函数，在脑海里使劲寻觅了一番，发现没有之后，就开始想到用正则表达式的.test()方法：reg.test(str);

    当str字符中匹配到正则表达式reg时就返回true,否则就返回false.

然后就开始想用什么正则表达式，判断英文字符简单，以前有学到过/.*[a-z A-Z]/表示匹配字符串中出现的英文大小写字母的字符。
可是中文字符是什么鬼？于是就google了一下，原来使用/[/u4e00-\u9fa5]/就可以匹配字符串中出现的中文字符。
可是如果写成/[a-z A-Z \u4e00-\u9fa5]/.test(str)只能检测到字符串中是否含有符合要求的字符串，可是题目的要求是字符串中必须全是符合要求的，那怎么办呢？
这时候只要反转一下，我们判断字符串中是否含有不符合要求的字符不就OK了吗？于是要求1就顺利可以完成了。

而题目相对就简单多了，只需要使用/\D/就可以匹配字符串中出现的非数字字符

关于《海边的卡夫卡》

星期天为什么是星期天？仅仅只是因为我们赋予了它特殊的含义，而它本身作为时间洪流中的24小时，与其他时间并无本质差别。可它最终还是被赋予了某些意义，特别是在一些在意它的人眼中，比如说我。

今天刻意地没有早起，无论精神上还是身体上都不想早起，于是就直接忽视了闹钟，一觉睡到自然醒。这在昨天就已经想好了合适的借口，“明天就用来干一些杂事放松一下自己吧”。于是9点洗漱完，背了半小时的单词，然后开始看《海边的卡夫卡》，本想看半小时就做该做的事了，结果就停不下来了，或者说是我不想停下来去干别的事。一直看到吃午饭，洗完衣服后就接着看继续看，中途睡了一会，在吃晚饭前看完了余下的全部。下面是正经的随感了：

总体来说我对于这本书没有特别大的感触，故事发生得怪诞离奇，两条主线最后合二为一，然后离奇结尾。我大概也属于想象力匮乏的那类人，无法琢磨透小说中的世界，也无法琢磨透作者的想法。仅有一些模糊大概的想法。

  • 主人公1号：田村卡夫卡，15岁的少年无法忍受自己周围的环境，甚至厌恶自己的父亲，于是在准备好一切后孤身离家出走。尽管他试图挣脱困住自己的牢笼，可是他没有逃脱父亲给他下的诅咒，他间接杀死了父亲，他遇到了自己的生母，还和她交合，在梦里和姐姐交合（好像不是亲生的）。但最后他在另一个世界想通了，于是回到现实并勇敢接受了既定的事实，成为了可能是最顽强的15岁的少年。

    先为自己筑起厚厚的围墙，不愿让任何人进入自己世界。可是慢慢发现一些东西禁锢住了自己，让自己不再是自己，于是开始奋力抵抗

  • 主人公2号：中田，因为小时候的离奇遭遇，导致记忆丧失，并且连带着读写能力也丧失了。靠着亲人的微小帮助，政府的救济金，以及能和猫沟通的能力，一个人漫无目的地活到了50多岁。然后因为一次莫名其妙的杀人，突然觉醒一般，开始了自己的寻找之路，寻找自己失去的那一半影子。而在寻找过程中不自觉间改变了一个叫“星野君”的卡车司机的世界观。

    接受自己的缺失。

如果如书中所说，世界都是隐喻。那田村卡夫卡隐喻着什么呢？中田又隐喻着什么呢？整部小说又在隐喻着什么呢？ 可能要多看几遍才能慢慢明了吧，可是我又不想再看第二遍了。那我看书究竟是为了什么呢？

我们都在持续失去种种宝贵的东西，宝贵的机会和可能性，无法挽回的感情。这是生存的一个意义。但是我们的脑袋里有一个将这些作为记忆保存下来的小房间。而我们为了解自己的心的正确状态，必须不断制作那个房间用的检索卡。也需要清扫、换空气、给花瓶换水。换言之。你势必永远活在你自身的图书管里。
