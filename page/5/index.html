我的实习之路（六）

发表于 2016-07-31   |   分类于 工作   |     |   阅读次数

以下开头是废话

从6月20号到今天，已经实习一个多月了。而我也成功从一名愣头全青转变成为了愣头半青——已经渐渐适应了公司的环境、租房的环境、工作的方式以及和他人交流的方式。所以我想就目前自己的一些经历做点儿总结，希望能够帮到其他人和接下来的自己。

去年9月份，我站在人生的十字路口……算了，这个蛮有意思的往事还是下次再说吧~

这将会是一系列连载篇，大致顺序是：

简历+面试 -> 租房 -> 困惑 -> 融入 -> 观察与总结 -> 自我提高
如有跳篇，不要惊慌，我比较随意😁

以上开头是废话

六、自我提高篇

从初中开始，我才渐渐明白一个道理：

永远别把提高自己的希望寄托在其它的人或事物上

那时的我其实不是不想依靠外物，而是真的没有什么可以靠，于是就只能靠自己了~而靠着靠着竟也养成了习惯。

可是在找实习前，也就是在我自学前端近6个月之后，我居然开始幻想或者说迫切地希望能有个“师傅”来带我。因为我好像对自己失去了信心，我知道我在这个领域没有什么天赋加成，而且我不知道前面那条路的终点是什么，我甚至不知道下一个路口在哪里。

恐惧轻松地把我坚守近10年的信条撕成碎片，而这也是我想暑假找实习的部分原因——我真的太需要一个“师傅”来为我指明前方的路了。

好吧，说了这么多，其实就是四个字：“我迷茫了”。迷茫产生了恐惧，随后为了克服恐惧，我产生了不切实际的幻想。

很多时候我们产生的愿望或者说幻想，都来源于恐惧。因这能让我们完成“自我救赎”，有助于我们减轻痛苦、逃避现实和恐惧。

而我到工作一个月后才悟出这些，原因很简单——我的希望破灭了。

遥想我在面试时都会问：“实习生会不会有人带？”而HR都是非常肯定地回答：“我们公司有非常成熟的老人带新人计划，balabala……”我当然天真得就信了咯！于是就越发对“实习能够让我快速提高”这个想法充满信心。

然而没错！故事在我入职几天后产生了巨大的转折。我突然意识到：

  1. 你的“师傅”可能根本无法让你仰之弥高；
  2. 你的“师傅”可能根本不会先了解你目前的状态，了解你的潜力；
  3. 你的“师傅”可能根本不会为你的发展考虑，帮你突破瓶颈；
  4. 最悲哀的一点：你的“师傅”可能根本不会把你当成徒弟看。

以上只是我观察和领悟到的一些“有可能”发生的事实，也希望后来之人能够早些有点心理准备。而关于这个我也不会去抱怨什么，毕竟大家都是成年人了，你可能什么好处都不能带给你的“师傅”，他却还要迫于公司要求，花点心思在你身上，而你居然还要他为你掏心掏肺，劳神费力？What are you thinking about?不带你是本分，带你是情分，而你们一开始有屁个情分，不要自作多情。

恩，认清现实，然后才好对症下药。于是我又重拾我当年的信条，而经过这一次的破碎重组，这个信条又坚韧不少。下面就是正题了：当我确定自己所要发展的领域之后，如何进行自我提高？

不不不，这个命题太大了！我自己还没提高多少，屁事都没干成，不能误人误己。因此我就说说我自己关于如何在实习的时候自我提高？的想法：

  1. 多看书
  2. 多提问
  3. 多总结

这三个好像是老生常谈的话题，不过我稍微结合自己的经历具体讲一下。

多看书？

首先关于这个“书”，当然只是一个抽象的概念，它可以指实体书，可以是电子书，也可以是别人的一篇文章，还可以是一些简短的问答，甚至是你周围优秀的人的一些好习惯。

  • 关于实体书：你所在领域的经典名著、能够为你答疑解惑的文学类/哲学类名著、能够让你认清现实的经济类/说教类/管理类名著，都是需要看的。至于先后顺序的话，取决于你的现状，哪里不够补哪里~如果恰好都不够的话，那就只能先救近火了~
  • 关于电子书：可以尝试去找实体书还没出版的原著，如果是IT类，可以去找一些流行技术的官方文档。
  • 关于文章：可以找有兴趣的所在领域比较有名的人的博客、微博或者公众号，有事没事刷一刷。
  • 关于问答：可以逛逛自己所在领域一些有名的论坛，就和逛贴吧差不多(＾－＾)。
  • 关于优秀的人：优秀的人往往有许多优秀的习惯，你可以仔细观察他上班后的一举一动，然后列出他从上班到下班的时间里都做了什么，从中挑出你认为是好的习惯，努力模仿吧~（优秀的艺术家模仿，伟大的艺术家偷窃~）

多提问？

先去看一遍提问的智慧，当然这个原版的排版实在太烂了，建议看中文翻译版，大概半小时就能看完。现在互联网这么强大，基本上已经从以前的“勤学好问”变成现在的“勤学好搜”了~

虽然说不要依赖你的“师傅”，不要幻想他能给你多大帮助，但是碰到一些无法解决的问题当然还是要“无耻上问”的(～￣▽￣)～多问一些有涵养的问题，可以促进师徒感情升温哦~

多总结？

总结是必须的，当然如果你对自己的记忆很有信心的话……不好意思，没有用！也还是要总结╰(°▽°)╯。

总结的时候能够让你看清一些事情的本质，有助于加深印象，balabala……

好吧，既然总结有这么多好处，那我应该怎么总结呢？
嚯！不是吧？这么快就忘了？前面说好的提问的智慧呢？(°o°；)

我只能说说我的，因为我字比较烂不想写纸上，不然写纸上体验肯定更加好点~所以如果你和我一样不想写纸上的话，建议先花20分钟学下Markdown。然后找个自己喜欢的markdow编辑器，简书、锤子便签、作业部落什么的都还不错。我用的是作业部落~

如果你不在意排版的话，可以用有道云笔记或者印象笔记，我用的是印象笔记，很好很强大哦~

The last but the most important？

哦，我好像漏了最重要的一点，你得有这个意愿，并且马上开始行动！空想好像没什么用……

自我提高的总结：

  1. 不要把希望寄托在其它人或事上
  2. 很多时候我们产生的愿望或者说幻想，都来源于恐惧。
  3. 用正确的姿势进行自我提高：多看书、多提问、多总结。
  4. Just do it !

我的实习之路（一）

发表于 2016-07-28   |   分类于 工作   |     |   阅读次数

以下开头是废话

从6月20号到今天，已经实习一个多月了。而我也成功从一名愣头全青转变成为了愣头半青——已经渐渐适应了公司的环境、租房的环境、工作的方式以及和他人交流的方式。所以我想就目前自己的一些经历做点儿总结，希望能够帮到其他人和接下来的自己。

去年9月份，我站在人生的十字路口……算了，这个蛮有意思的往事还是下次再说吧~

这将会是一系列连载篇，大致顺序是：

简历+面试 -> 租房 -> 困惑 -> 融入 -> 观察与总结 -> 自我提高
如有跳篇，不要惊慌，我比较随意😁

以上开头是废话

一、简历和面试篇

一开始把这步想得很简单，「不就是做个简历，然后随便找公司就开始投嘛」，可是事实是：没那么简单，就能找到，聊得来的伴……

1. 下载简历模板

本来想自己做的，可是尝试做的时候发现「word真的不适合用来做设计」，各种恶心。可是如果改用PPT和PS做的话又只能导出成图片或者PDF，万一有些变态公司要word呢？

于是就只能去网上开始找模板，经历各种折磨后，我发现了一个简历样式丰富、设计精美，基本满足各种刚需的网站五百丁 。你可以选择免费在线制作，完成后还可以生成成一个网址（如果自己把网址生成二维码，贴到简历上，瞬间逼格满满~），最后还可以免费导出成图片或者PDF。

可是还是这无法满足我（欲求不满？），我发现在「五百丁」还是大量便宜又好看的付费模板下载，而且一买就是三种配色方案! 没错，就是3份Word！这钱花的绝对值啊！还顺便支持了我国的正版大业！而且网站上还有大量矢量格式的小图标，让你修改简历又方便了很多！（当然如果你又想质量好，又想不花钱的话，那你赶紧去找！找到告诉我……）

最后我还把它安利给了身边的其他人，要是也有人早点安利给我多好！？

2. 组织结构和内容

既然搞定好了一件华丽的服装，你总得给她塞点人模狗样的东西进去吧？别人虽然会被好看的外表吸引，但你内在不过关，人家不敢要你呀~

先想好把简历分成几大块内容（搭好框架），就和造人一样一样的~你总得有头、有四肢（五肢？）和躯干吧！我的框架是「个人信息」、「专业技能」、「兴趣爱好」、「前端相关经历」、「校园经历」、「教育背景」和「个人介绍」七部分，当然必须主次分明，像「兴趣爱好」我就只用了4个小图标~

然后就开始往框架里填东西了（人造肉工程？）。对于每个版块，把自己能想到所有的东西掏出来写在纸上，然后根据需要，进行有效筛选（建议把自己想成一个HR），因为就一张A4纸大小，你还想写论文啊？

这个内容就看个人本事了，如果你大学牛逼闪闪，应该很容易就为框架充入强健的肌肉；如果你大学平平凡凡，那就充入平凡的肌肉嘛~什么？你说什么都没有？XX说：「每个人都有闪光点，要善于发现……」唉~那就充个鸭肉进去？

到了写简历的时候才发现，我的大学原来是这样度过的……

建议：简历内容不要掺假，最好连水分也不要掺，不然到时候一问就会露陷。

3. 确定期望的地方、薪资和公司

好了，前戏终于做足了，下面按道理应该要进入高潮了~屁！想得太美！还有个中戏好不……

忙活半天，才特么把简历搞定（建议还要再反复修改或者找别人挑刺），心想终于可以愉快地去投简历了~不不不，投简历前，你得想清楚一些事情，以便于你能够“高逼格”地投简历，而不是看到一块肉就想上去咬一口，比如吃蚊子和狮子？（拜托~请尊重蚊子的生命和自己的生命好吗？）

我当时能想到的几个条件就是「工作地范围」、「薪资范围」和「理想公司」，当然你觉得自己可以身兼多职，那你还得考虑下「职位」。想清楚这些以后，去招聘网站投简历的时候就可以很方便地筛选出有效信息。

所以你得根据自身条件，想清楚到底咬哪块肉，这样不至于发生惨案~（当然人家最后愿不愿意割肉喂你，就看你本事了。）

4. 投简历

报告长官！武器装备已准备！射击范围已确定！请指示作战区域！

各部队请注意！作战区域为各大招聘网站，不嫌烦的请前往「58同城」和「赶集网」，对设计和用户友好要求高的请前往「拉勾网」和「实习僧」,已经确定想去公司的请前往「公司官网」！指示完毕！出发！

这里重点安利下拉勾网，注册后不用在它的网站上再编辑一遍自己的简历，只要上传已经准备好的文件即可！而且，如果绑定了邮箱，反馈很及时！不至于石沉大海……

5. 做面试准备

投了N份简历后（不建议一次投太多），就可以休息了？想来个葛优躺？还早着呢……

  • 还要为马上开始的面试做准备~（PS：入职技术类的，还要为笔试做准备~）
  • 自我介绍必须有！最好有逐字稿，能背到不像在背稿子~（夸张点的还要注意自己的表情和眼神。）这是我自己总结的写自我介绍的正确姿势。
  • 常见问题要准备好~（你为什么选择我们公司？你的职业规划？）
  • 变态问题也可以准备下~（如果让你给自己画一幅画，你会画什么？）
  • 群面技巧也要准备~（万一要群面呢？）

6. 安排好面试行程

现在已经磨刀霍霍向猪羊了，就等面试通知了！这里可能会有人问，万一我投的简历都没有经过筛选怎么办？那我不是白准备了？

拜托！能不能自信点？而且就算一开始没有面试机会，那也不算白准备，只要你没烂到一定境界，总有愿意吃你这颗草儿的马。（不要去假设一件事情还没发生就注定会失败，那样的人生挺痛苦的~）

那现在就假设你已经收到通知了，一般招聘网站给你发邮件后，HR会给你打电话。在这中间，自己得先确定接下来的几天或者一礼拜，能抽出去面试的日子。下面就等电话约时间地点即可~（注意：最后记得问下去面试需要带的东西~）

7. 临场发挥

你风尘仆仆地赶到了面试的地方，填了一张表，还可能做了一份笔试题，然后就开始面试了~

第一次的时候，紧张在所难免（我第一次时连自我介绍都抖音），保持微笑，注意眼神交流。等做完自我介绍，就兵来将挡水来土掩了，而面试官一般会针对你的简历进行“详细交流”（所以我前面说最好不要掺水），然后会问一些其他问题~

这里临场发挥真的很重要，有些问题确实会是你没有准备到的，用清晰的逻辑+诚恳的语气回答就好了~
这是我第一次的面试经历

8. 仔细观察

从你进入公司那一刻起，你就应该仔细去观察一些东西了。比如说我关注的点：妹子是否多？质量是否高？办公环境（装修）如何？员工神态气质总体如何……

9. 决定入职

如果没有什么意外，在经过一轮或者多轮面试后，HR就会打电话通知你是否通过他们的考核。而一旦他们决定给你offer，此时主动权就在你手上了，你可以决定去或者不去。（选择反转）

所以如果你在他们要求报到的日子前，拿到几个offer的话，你就可以从中筛选出最适合自己的那一个。

10. planB

最后的最后，如果万一面试了几个，可是拿到offer后觉得不满意,或者没有拿到offer，或者甚至连面试机会都没有，那怎么办？我只能说说我的经历，经供参考。

因为我实际是从6月份才开始投简历的（虽然4月份就做好了简历，但是没敢投。因为那时候觉得自己底子太薄，肯定没希望），而且还要为20多号开始的期末考做准备，导致我时间严重紧张。基本上就是：白天在图书馆复习，晚上回寝室。然后睡觉前看下招聘网站，筛选出心仪的几个公司投下简历。

因为投完简历后一般要等下礼拜才会通知你面试，而我每礼拜只投3、4份简历，所以我前后一共只参加了两次面试。

当时的我是这么想的：

如果能进那就最好了，如果不能进那就说明自己的基础还是不够。那就暑假回家补，然后等秋招呗~

总结：

  • 上五百丁买模版
  • 填好简历
  • 有目的地开始投递
  • 准备面试
  • 选择入职
  • plan B

多个单词可以考虑用连字符-，因为比驼峰命名更加清晰。

id和class哪家强

一般原则是，类应该应用于概念上相似的元素，这些元素可以出现在同一页面上的多个位置，而ID则应该应用于不同的唯一元素。只有在绝对确定这个元素只会出现一次的情况下，才应该使用ID。如果你认为以后可能需要相似的元素，就使用类。

不要过度使用类，会在文档中添加无意义的代码。
不要过度使用div,这是代码结构不合理且过分复杂的一个信号。

程序员、内容管理员和其他前端开发人员可能需要了解代码的元素和设计是如何工作的，而建立样式规范是维护或实现站点的好方法。可以确保网站开发以可控的方式进行，同时防止样式随着时间的推移变得散乱。还非常有助于向新职员或承包商介绍他们不熟悉、可能是很复杂的系统。

  1. 静态样式规范：一个文档、网页或小型站点
    解释代码和站点的视觉设计是如何组合在一起的。
    • 最基本的样式规范应该描述一般性设计原则，如如何处理标题和其他排版元素、网格结构的设计方式是什么，以及如何使用哪个调色板。
    • 好的样式规范还应描述文章、新闻条目和通知等重复元素的处理方法，决定他们应该如何实现和不应该如何实现。
    • 更详细的样式规范可以包含编码标准，使用的HTML/CSS版本、选择的可访问性级别、浏览器支持细节和一般的编码最佳实践。
  2. 动态样式规范
    因为及时更新静态样式规范需要花费大量精力，所以它们常常很快就与当前网站不同步了。所以使用动态的样式规范（“模式组合”）

模式组合是一个页面或一系列页面，它们使用当前的样式表显示站点上可能出现的每种样式排列组合，从标题级别和文本样式直到特定的内容和布局类型。

友情、爱情和亲情之友情

发表于 2016-07-18   |   分类于 日记   |     |   阅读次数

小学时代

那时候的我在学校里几乎没什么朋友，何况那时我也不懂朋友是什么概念，就只是大家好了就一起玩，不好了就互不理睬。不过想起那时候拉帮结派的日子，觉得有点搞笑和伤感。

那时候的我倒是和附近邻居家的孩子玩得很欢，特别是每年的寒暑假。但可能是家庭带来的自卑感，我始终没有融入到他们中去过。（这可能也是我至今几乎没什么朋友，也不太会放开自己的根本原因。）

初中时代

初中整个三年倒是有一个我自认为最要好的朋友，反正我是把他当最好的朋友，不过他好像和谁都是好朋友。我初三暑假还睡过他家一段时间， 不过高中以后就几乎断了联系……大学以后更是没有再联系过。

高中时代

高中开始，我的自卑感开始渐渐消退，因为我莫名其妙的就想开了。那时候我也有一个我自认为最要好的朋友，但他好像还是和谁都是好朋友。他影响改变了我许多，至少让我变得更加开放。但是大学以后，我俩几乎不怎么联系，因为有时候聊起来竟然不知道聊什么。不过我想我们俩当谁有困难的时候，另一方一定会帮忙，至少我是会这样的。

大学时代

到了大学，我还是只有一个我自认为最要好的朋友。可我不知道他是否认为我是他最好的朋友。可能是性格什么的已经养成，这时候能凑在一起的，双方都不会觉得无聊。我和他聊的时候很自在。放得很开，几乎没什么可以顾忌的。

其实大学里还有一个我很想和他成为朋友的人，但是由于接触机会真的太少太少，相互袒露心扉的机会也几乎没有，也算是一大遗憾。

总结

也就是说至今为止，我的生命里有且只有三个，我认为是朋友的男人，而且其中一个已经完全失联，第二个处于半失联状态。最后我发现一个奇怪的共同点：

我和他们都在一张床睡过！

除了我爸、我哥和他们仨，我好像真的就再也没有和别人两个人一起睡过了。

我对朋友的定义：首先你必须得有和他聊天欲望，其次三观起码要一致，最后你觉得当你遇到困难的时候，他肯定会伸出手帮你。甚至，说得夸张点就是如果你遇到危险，他会冒险把你藏他家里。

最后说到女性朋友的话，目前聊得起来的有且仅有三个，而且都是高中同学。因为大学里，我几乎没有和女生放开地聊过天，原因不详。

至此，便是我惨不忍睹的朋友篇。

早起日记-关于工作中遇到的前端坑

发表于 2016-07-15   |   分类于 工作   |     |   阅读次数

如果从6.21号入职开始算的话，我以实习生的身份进入公司已经有将近一个月的时间了。在这个5人前端组里，过得还是十分轻松愉快的。

  • 没有什么险恶的钩心斗角（当然也有可能是我还是个小不点），组里的人也都很好相处；
  • 正式员工也没有架子，有什么问题他们都会积极和你一起解决。

当然有利也有弊，轻松愉快的另一面就是：我有时会很闲，闲得自己在看文档学习甚至会打瞌睡，以致于让我觉得领工资受之有愧。

不过闲归闲，这礼拜阿冰师傅给了我两个简单的需要上线的活动页面让我独立完成，有问题找他。我当时有点兴奋，因为总算可以有点事情做了。虽然页面很简单，逻辑也很简单，但途中还是遇到了不少坑，也学到了不少东西，比如说：

  1. 如果要适配各种屏幕的手机，一般只能做到宽度自适应。（因为如果同时做到高度自适应的后果可能就是页面比例变形）。而这个时候，如果你制作的是一个单页的话（也就是说设计稿页面的高度恰好是一个屏幕的高度），
    • 在长宽比大的手机（like iphone 6plus）中，会在底下出现一块空白，
    • 在长宽比小的手机（like iphone 4）中，需要滑动才能看到剩余部分
  2. 如果在微信朋友圈打开分享链接，是无法直接唤起已有APP的。
    • 在iOS系统下，会跳转到APP Store，然后再按打开按钮，才能唤起APP；
    • 而在Android系统下，就比较恶心了。微信会先跳转到腾讯应用宝下载页面，然后如果你这时候点【腾讯应用宝安全打开】，它会自动下载腾讯应用宝；如果你点【普通打开】，它会提示在普通打开下会存在安全隐患，然后给你两个选项【继续】和【安全打开】，按照人类的逻辑，这时候我不管点哪一个都应该唤起APP。然而结果是，你如果点【安全打开】，又会进入下载。必须点继续才能唤起APP。
  3. 还有一些历史遗留问题，我使用了公司规范里的一些函数，结果有几个函数内部的地址不正确，导致在IOS和Android下结果不一样。后来我们找了半小时，然后问主管（我们的老大）洋哥，然后洋哥再找客户端开发人员，才找到问题所在。

对于问题1，目前只能通过给body设置一个背景色，让它和页面底部的颜色一致，使页面在大屏手机下不至于出现一块空白。而小屏手机页面无法完全展示，需要滑动才能看到其余部分的问题还是没法解决。【除非设计稿一开始的尺寸是按照小屏手机来设计的】。

不知不觉已经7月半了，我得加速看算法和数据结构才行了！

我究竟是不是有社交困难呢？应该不是……该有的情商我还是有的，一般的交际能力我也是有的，但是我却在自我抵触社交。比如说：

  • 作为一个实习生（愣头青），碰到男的就叫哥，碰到女的就叫姐，表面看来好像没什么错，而且这样既不会显得很谄媚，也不会落了礼貌和亲近。可是我心里真的很抵触，是因为自我清高在作祟？还是因为无法产生发自内心的尊敬？
  • 今天我们技术组和市场部聚餐的时候，我在想我到底要不要敬个酒呢？一直在纠结……最后还是放弃了，默默在角落开吃。遇到什么事也就附和地笑笑，基本没怎么讲话。

没想到连对别人称呼都能困扰我（如果都能直接叫名字该有多好……）。

所以，有以下想法，望我日后自己有一天能够带团队的时候能够谨记，而不是像鲁迅先生说的健忘的人一般，忘了自己曾经吃过的苦，还把这种苦让别人再尝受一遍。（这当然不是原话）

  1. 自己做团队建设的时候，首先解决称呼问题，这样不至于有太多的官僚气息也不会让新人困惑。
  2. 尽量解答新人各方面的问题，能让他畅所欲言（当然是先经过他自己观察思考的，而不是无脑问题）。
  3. 如果新人第一次聚会，一定不要让他尴尬得无所适从，主动介绍他，让他快速融入团队。
  4. 解决新人的培养模式问题，先探测底细，然后合理安排任务和进度，而不是不管他是谁，不管他的能力如何，都特么的从头来一遍，浪费大家的时间。
