数据结构和算法

大话数据结构学习笔记(4)_栈和队列

栈（stack）

科尔特手枪的弹夹，最先放进去的子弹，最后打出。

栈：是一个后进先出（Last in first out, LIFO）的线性表，它要求只在表尾进行删除和插入操作。 （对栈来说，这个表尾称为栈的栈顶（top），相应的表头称为栈底（bottom））

栈是一种重要的线性结构，先进后出。比如浏览器的后退，一些软件的撤销功能、C语言的函数。

栈的抽象数据类型

栈的顺序存储

栈的顺序存储也是线性表顺序存储的简化，简称为顺序栈。也用数组实现，讲数组下标为0的一端作为栈底。

用 top 变量指示栈顶元素在数组中的位置。当栈存在一个元素时，top 为0；为空栈时，top为1

  • 进栈

  • 出栈

    ​

  • 缺点：必须事先确定数组存储空间的大小，万一不够用时，需要手动扩展数组容量。

两栈共享空间（类似于合租）：用一个数组存储两个相同类型的栈。让一个栈的栈底为数组的始端，即下标为0处；另一个为数组的末端，即下标为 arr.length-1处。两个栈如果增加元素，就是两端点向中间延伸。（适合两个栈的空间需求有相反关系时）

实践

利用栈的数据结构特点，将二进制转换为十进制数

栈的链式存储

简称栈链，较好的方法就是将栈顶放在单链表的头部，栈顶指针和单链表的头指针合二为一，不需要头结点。

  • 进栈 O(1)

  • 出栈 O(1)

  • 优点：基本不存在栈满的情况

顺序栈 VS 栈链

顺序栈

  • 需事先确定一固定长度，可能存在空间浪费的问题
  • 存取时定位很方便

栈链

  • 每个元素有指针域，增加内存开销
  • 长度无限制

栈的引入简化了程序设计问题，划分了不同关注层次，使得思考范围缩小，更加聚焦于待解决问题核心。无需像数组等需考虑数组下标增减等细节问题。

实践

  • 递归：把一个直接调用自己或通过一系列的调用语句间接地调用自己的函数，称做递归函数。（编译器使用栈实现递归）

  • 不需要括号的后缀表达式——逆波兰表达式（RPN），(1-2)(4+5) -> 1 2 - 4 5 +

    利用逆波兰表达式和栈，可以让计算机快速计算出运算结果。遇到数字入栈，遇到操作符出栈，并进行计算结果，然后入栈。

    中缀表达式转换为后缀表达式：从左到右遍历中缀表达式的每个数字和符号，若是数字则直接输出，即成为后缀表达式的一部分；若是符号，则判断其与栈顶符号的优先级，若该符号是右括号或者优先级低于栈顶符号，则栈顶元素依次出栈并输出，直到遇到左括号或栈空，才将该符号入栈。（* / ( 直接入栈）

队列（queue）

操作系统和客服系统中，都应用了队列的先进先出排队功能。

队列：是一种先进先出的线性表（first in first out, FIFO），同样需要顺序表或链表作为基础。只允许在一端进行插入操作，而在另一端进行删除操作的线性表。（允许插入的一端称为队尾，允许删除的一端称为队头。）

如，输入缓冲区接受接盘的输入就是按队列的形式输入和输出的。

队列的抽象数据类型

队列的顺序存储

  • 入队列 O(1)
  • 出队列 O(n)

为什么出队列时一定要全部移动呢？如果队头不需要一定在下标为0的位置，出队的性能就会大大增加。使用front指针指向队头元素，rear指针指向队尾元素的下一个位置，当front等于rear时，队列为空队列。但，即使是这样改进之后，还是会存在“假溢出的问题”。

循环队列

解决假溢出。只需要灵活改变front和rear指针即可。也就是让 front 或 rear 指针不断加 1 ，即使超出了地址范围，也会自动从头开始。可采取 取模运算处理： (rear+1)%qsize

此时队列满的条件是： (rear+1) % Qsize == front

通用的计算队列长度公式：（rear - front + Qsize）% Qsize

  • 初始化

  • 入队列 O(1)

  • 出队列 O(1)

队列的链式存储

和栈相反的是，队列一般用链表实现，称为链队列。队头指针指向链队列的头结点，队尾指针指向终端节点。空队列时，两者都指向头结点。

  • 新建

  • 入队列

  • 出队列

  • 销毁队列

循环队列 VS 链队列

时间上：都为O(1)。

  • 循环队列事先申请好空间，试用期间释放
  • 链队列的每次申请和释放节点也会存在时间开销

空间上：

  • 循环队列必须有一个固定的长度，可能存在存储元素个数和空间浪费的问题
  • 链队列只需要一个指针域，更加灵活。

总的来说，在可以确定队列长度最大值的情况下，建议使用循环队列。否则使用链队列。

递归和分治

递归定义：直接调用自己或通过一系列的调用语句间接地调用自己的函数，称为递归函数。注意必须有终止条件。

  • 效率比较低，尽量用迭代。
  • 迭代使用的是循环结构，而递归使用的是选择结构。使用递归能使程序的结构更清晰、简洁、易懂。
  • 递归函数分为调用和回退阶段，递归的回退顺序是它调用顺序的逆序。

斐波那契数列、阶乘，将任意长度的输入字符串反向输出的递归实现。

分治：当一个问题规模较大且不易求解的时候，可以考虑将问题分成几个小的模块，逐一解决。

用分治思想处理问题，其各个小模块通常具有与大问题相同的结构，这种特性使递归技术有了用武之地。
