算法综述

概念

算法：是解决特定问题的求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示一个或多个操作。（泡妞的技巧和方式，对于给定的问题，是以可以有多种算法解决的）

指令：能被人或机器等计算装置执行。

算法特征

  1. 输入：>=0
  2. 输出：>=1
  3. 有穷性：指算法在执行有限的步骤之后，自动结束而不会出现无限循环，并且每个步骤在可接受的时间内完成。（一定要可接收啊～）
  4. 确定性：每个步骤都具有确定的含义，无二义性；在一定条件下只有一条执行路径，相同的输入只能有唯一的输出结果；每个步骤都应该被精确定义而无歧义。
  5. 可行性： 每一步都必须是可行的，即每一步都能够通过执行有限次数完成。

算法设计要求

  1. 正确性：指算法至少应该具有五个基本特征
    • 无语法错误
    • 对合法输入能产生满足要求的输出
    • 对非法输入能够产生满足规格的说明
    • 对故意刁难的测试输入都有满足要求的输出结果
  2. 可读性：便于阅读、理解和交流
  3. 健壮性：当输入不合法时，算法也能做相关处理，而不是产生异常、崩溃等结果。
  4. 时间效率高和存储量低

效率度量方法

  • 事后统计方法：通过设计好的测试程序和数据，利用计算机计时器对不同算法编制的程序的运行时间进行比较，从而确定算法效率的高低。（必须事先编好测试测序、测试环境差别、测试数据设计困难）——事后诸葛亮
  • 事前分析估算方法：在计算机程序编写前，依据统计方法对算法进行估算。

影响高级语言编写的程序在计算机上运行时所消耗的时间的因素：

  1. 算法采用的策略，方案
  2. 编译产生的代码质量（软件）
  3. 问题的输入规模
  4. 计算机执行指令的速度（硬件）

抛开这些和计算机硬件、软件（编译器）有关的因素，一个程序的运行时间依赖于算法的好坏和问题的输入规模。

研究算法的复杂度，侧重的是研究算法随输入规模扩大增长量的一个抽象，而不是精确地定位需要执行多少次，因为这样的话，又要考虑回编译器优化的问题。

所以不关心编程语言、运行环境，只关心所实现的算法。因此，不计循环索引的递增和循环终止条件、变量声明、打印结果等操作。最终，在分析程序的运行时间时，最重要的是把程序看成是独立于程序设计语言的算法或一系列步骤。

在分析一个算法的运行时间时，重要的是把基本操作的数量和输入模式关联起来，也就是基本操作的数量必须表示成输入规模的函数。

函数的渐进增长：给定来那个函数f(n)和g(n)，如果存在一个整数 N，使得对于所有的 n > N，f(n) 总是比 g(n)大，那么，我们说 f(n) 的增长渐近快于 g(n)。

结论：判断一个算法的效率时，函数中的常数和其他次要常数常常可以忽略（只关注最高次项），更应该关注主项（最高项）的阶数。

判断一个算法的好坏，大量数据。某个算法，随着n的增大，它会越来越优于另一算法，或者越来越差于另一算法。

算法复杂度

算法时间复杂度：在进行算法分析时，语句总的执行次数 T(n) 是关于问题规模 n 的函数，进而分析 T(n) 随 n 的变化情况并确定 T(n) 的数量级。 算法的时间复杂度，就是算法的时间量度，记作： T(n) = O( f(n) ) 。它表示随问题规模 n 的增大，算法执行时间的增长率和 f(n) 的增长率相同，称作算法的渐进时间复杂度，简称为时间复杂度。其中 f(n) 是问题规模 n 的某个函数。

  • 用大写 O() 来体现算法时间复杂度的记法，称之为大 O 记法。
  • 一般情况下，随着输入规模 n 的增大， T (n) 增长最慢的算法为最优算法。

（算法的分析，一般在没有特殊说明的情况下，都是指最坏时间复杂度。）

推导大O阶：

  • 用常数1取代运行时间中的所有加法常数
  • 在修改后的运行次数函数中，只保留最高阶项
  • 如果最高阶项存在且不是1，则去除与这个项相乘的常数。
  • 得到的最后结果就是大O阶

常数阶（O(1)）、线性阶（O(n)）、平方阶（O(n^2)）、n次方阶、对数阶（出现在while中 O(logn)）。循环的时间复杂度等于循环体的复杂度乘以该循环运行的次数。函数调用的时间复杂度分析

常用时间复杂度所耗时间排序：

O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)

空间复杂度：程序运行所消耗的内存空间度量

时间和空间可以互换（闰年）
