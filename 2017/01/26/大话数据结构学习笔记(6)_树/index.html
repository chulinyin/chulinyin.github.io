大话数据结构学习笔记(6)_树

发表于 2017-01-26   |   分类于 数据结构和算法   |     |   阅读次数

树（Tree）

一对多的数据结构。

定义

是 n(n>=0) 个节点的有限集。当 n=0 时，称为空树，在任意一颗非空树中：

  • 有且只有一个特定的称为根 (Root) 的结点；
  • 当 n>1 时，其余节点可分别为 m(m>0) 个互不相交的有限集 T1 、T2 、… Tm ，其中每个集合本身又是一棵树，并且称为根的子树 (subTree) 。
结点分类

结点拥有的子树数量称为节点的度 (dgree) ，树的度取树内各结点的度的最大值。

  • 度为0的节点称为叶结点 (Leaf) 或终端结点；
  • 度不为0度结点称为分支结点或非终端结点，除根结点外，分支结点也称为内部结点。
结点间关系
  • 结点的子树的根称为结点的孩子 (child) ，相应的，该结点称为孩子的双亲 (parent) ，同一双亲的孩子之间互称为兄弟 (sibling) 。
  • 结点的祖先是从根到该结点所经分支上的所有结点。
  • 以某结点为根的子树中的任一结点都称为该结点的子孙。
结点的层次
  • 结点的层次 (level) 从根开始定义，根为第一层，根的孩子为第二层。
  • 其双亲在同一层的结点互为堂兄弟。
  • 树中结点的最大层称为树的深度 (Depth) 或高度。
其他概念
  • 如果将树中结点的各子树看成从左至右是有次序的，不能互换的，则称该树为有序树，否则称为无序树。
  • 森林 (Forest) 是 m(m>=0)棵互不相交的树的集合。对树中每个结点而言，其子树的集合即为森林。

树的存储结构

双亲表示法：以双亲作为索引的关键词的一种存储方式。假设用一组连续空间，存储树的结点，同时在每个结点中，附设一个指示其双亲结点在数组中位置的元素。（即每个结点不仅知道自己是谁，还知道它爸妈在哪里。即data是数据域，存储节点的数据信息。而parent是指针域，存储该节点的双亲在数组中的下标）

  • 可以根据某结点的 parent 指针找到它的双亲结点，所用的时间复杂度为 O(1) ，索引到 parent 的值为 -1 时，表示找到了树结点的根。
  • 但如果要知道某结点的孩子是什么，需要遍历整个树结构。

改进版双亲表示法：每个结点不仅知道自己是谁，他爸妈在哪里，还要知道他孩子在哪里，甚至他兄弟在哪里。

孩子表示法：由于每个结点可能有多棵子树，可以考虑用多重链表来实现。即每个结点有多个指针域，其中每个指针指向一棵子树的结点。有下面几种方案：

  1. 根据树的度为每个结点分配空间（但会造成空间的浪费）
  2. 根据每个结点的度为每个结点分配空间，专门取一位置来存储结点指针域的个数。（虽然节省了空间，但是维护难度太大）
  3. 顺序存储和链式存储结合，即数组和链表结合：用一个数组存放所有结点，每个结点有一个指针，指向它的第一个child 结点，该 child 结点包含 child 结点的下标和指向下一个 child 结点的指针。直到没有下一个 child 为止，则指针为空指针。（即，A.next = child1, child1.data=child1Index, child1.next = chidl2, … childn.data = childnIndex, childn.next = null）（但是这种方法只找到孩子，找不到双亲。）

双亲孩子表示法：在孩子表示法的基础上，再在数组每个结点上存储其 parent 在数组中的索引即可。根结点的 parent 的索引为 -1。

孩子兄弟表示法：任意一棵树，它的结点的第一个孩子如果存在就是唯一的，它的右兄弟如果存在也是唯一的。因此，可以为每个结点设置两个指针，分别指向第一个孩子，和该节点的右兄弟。（将一棵复杂的树变成了一颗二叉树。）

二叉树 (Binary Tree)

是 n(n>=0) 个结点的有限集合，该集合或者为空集（空二叉树），或者由一个根结点和两棵互不相交的、分别为根结点的左子树和右子树的二叉树组成。

特点
  • 每个结点最多有两棵子树，所以二叉树中不存在度大于2的结点。（注意，是最多，也就是说可以没有子树或者只有一棵子树。）
  • 左右子树是有顺序的，次序不能颠倒。
  • 即使树中某结点只有一棵子树，也要区分它是左子树还是右子树
五种基本形态
  1. 空二叉树
  2. 只有一个根结点
  3. 根结点只有左子树
  4. 根结点只有右子树
  5. 根结点有左右子树

若只从形态上考虑，只有三个结点的普通树只有两种情况：两层或三层。但是二叉树是区分左右的，因此有五种情况。

特殊二叉树

斜树：要么一路左斜，要么一路右斜。

满二叉树：在一棵二叉树中所有分支结点都存在左子树和右子树，并且所有叶子都在同一层上。

  • 叶子只能出现在最后一层
  • 非叶子结点的度一定是2
  • 在同样深度的二叉树中，满二叉树的结点个树一定最多，同时叶子也是最多。

完全二叉树：对一棵具有n个结点的二叉树按层序编号，编号为i(1<=i<=n)的结点与同样深度的满二叉树中编号为i的结点位置完全相同。（ 完全二叉树包含满二叉树。）

  • 叶子只能出现在最下两层
  • 最下层的叶子一定集中在左部连续位置
  • 倒数第二层，若有叶子结点，一定都在右部连续位置
  • 如果结点度为1，则该结点只有左孩子
  • 同样结点度二叉树，完全二叉树的深度最小
二叉树的特性

  • 第 i 层上至多有 2^(i-1)个结点

  • 深度为 k 的二叉树至多有 (2^k) - 1 个结点(k>=1)

  • 对任何一棵二叉树 T ，如果其终端结点（叶子节点）数为 n0 ，度为2的结点数为 n2 ，则 no = n2+1。

  • 具有 n 个结点的完全二叉树的深度为 :
    $$
    [log_2n] + 1
    $$

  • 如果对一棵有n个结点的完全二叉树的结点按层序编号，对任一结点i（1<=i<=n）有：

    1. 如果i=1，则结点i是二叉树的根，无双亲；如果i>1，则其双亲是结点[i/2]。
    2. 如果2i>n，则结点i无左孩子（结点i为叶子节点）；否则其左孩子是结点2i。
    3. 如果2i+1>n，则结点i无右孩子；否则其右孩子是结点2i+1。

    因为在完全二叉树中，结点i的左孩子是结点2i，右孩子为2i+1，因此以上结论很好证明。

二叉树的存储结构

  1. 二叉树的顺序存储结构：

    • 用一维数组存储完全二叉树中的各个结点，并且结点的存储位置能体现结点之间的逻辑关系。
    • 对于一般二叉树，尽管层序编号不能反映逻辑关系，但是也可以按照完全二叉树编号方式修改，把不存在的结点用 ^ 代替即可。

    但是极端情况，如斜树时，这样存储会造成巨大的空间浪费。

  1. 二叉树的链式存储结构：

    1个数据域 + 2个指针域 = 二叉链表

二叉树的遍历

从根结点出发，按照某种次序依次访问二叉树中所有结点，使得每个结点被访问一次，且仅被访问一次。

二叉树的遍历次序不同于线性结构，线性结构最多也就是分为顺序、循环、双向等简单的遍历方式。而树的结点之间不存在唯一的前驱和后继这样的关系，在访问一个结点后，下一个被访问的结点面临着不同的选择。

  1. 前序遍历

    若二叉树为空，则空操作返回，否则先访问根结点，然后前序遍历左子树，再前序遍历右子树。

  2. 中序遍历

    若树为空，则空操作返回，否则从根结点开始（注意并不是先访问根结点），中序遍历根结点的左子树，然后时访问根结点，最后中序遍历右子树。

  3. 后序遍历

    若树为空，则空操作返回，否则从左到右先叶子后结点的方式遍历访问左右子树，最后访问根结点。（这难道不就是css选择器匹配DOM树中结点时，所使用的方法吗？CSS的选择器是层序遍历还是前序遍历呢）

  4. 层序遍历

    若树为空，则空操作返回，否则从上到下，从左到右遍历访问结点。

已知前序和中序遍历或后序和中序遍历，可以唯一确定一颗二叉树。

线索二叉树

指向前驱和后继的指针称为线索，加上线索的二叉链表称为线索链表，相应的二叉树就称为线索二叉树（Threaded Binary Tree）。（其实就等于把一棵二叉树转变成了一个双向链表，便于插入、删除、查找结点）

利用中序遍历后的结点顺序。

对每个结点的结构加上一点扩容： lchild ltag data rtag rchild

  • ltag 为0时，指向该结点的左孩子，为1时指向该结点的前驱
  • rtag为0时，指向该结点的右孩子，为1时指向该结点的后继

优点

  • 充分利用了空指针域的空间
  • 保证了一次遍历就可以终身受用前驱后继信息

在实际问题中，如果所用的二叉树需经常遍历或查找结点时需要某种遍历序列中的前驱和后继，那么采用线索二叉链表的存储结构是很不错的选择。

树、森林及二叉树的相互转换

普通树转换为二叉树：

  1. 在树中所有兄弟结点之间加一连线。（有同一个parents的结点即为兄弟结点。）
  2. 对每个结点，除了保留与其长子的连线外，去掉该结点与其他孩子的连线。
  3. 调整位置，即可变成一棵二叉树。（根结点只有左子树）

森林到二叉树的转换：（若干棵树即可形成森林）

  1. 先将森林中的每棵树变为二叉树
  2. 第一棵二叉树不动，从第二棵二叉树开始，依次把后一棵二叉树的根节点作为前一棵二叉树的根节点的右孩子，用线连接起来。当所有的二叉树连接起来后就得到了由森林转换来的二叉树。

二叉树到树、森林的转换：

  1. 若结点 x 是其双亲 y 的左孩子，则把 x 的右孩子，右孩子的右孩子，……，都与 y 用连线连起来。
  2. 去掉所有双亲到右孩子之间的连线。

判断一棵二叉树能够转换成一棵树还是森林，标准是：如果该二叉树的根结点有右孩子，则转换为森林，否则转换为一棵树。

树与森林的遍历

树的遍历：

  • 先根遍历：先访问树的根结点，然后再一次先根遍历根的每棵子树。
  • 后根遍历：先依次遍历每棵子树，然后再访问根结点。

森林的遍历：前序遍历和后序遍历，其实就是按照树的先根遍历和后根遍历依次访问森林的每一棵树。

发现：树、森林的前根（序）遍历和二叉树的前序遍历结果相同，树、森林的后跟（序）遍历和二叉树中的中序遍历结果相同。

赫夫曼树

类似于课程成绩根据实际分数，判断优、良、及格、不及格，但是因为大多数人都是良好，如果从60开始划分等级，效率就比较低，从90开始向下判断效率就比较高。

定义与原理

带权路径长度 WPL 最小的二叉树称为赫夫曼树，也称为二叉树。

结点的路径长度：从根结点到该结点的路径上的连接数。

树的路径长度：树中每个叶子结点的路径长度之和。

结点带权路径长度：结点的路径长度与结点权值的乘积。

树的带权路径长度：WPL(weighted path length)是树中所有叶子结点的带权路径长度之和。

WPL 的值越小，说明构造出来的二叉树性能越优。那如何由森林构造最优二叉树（赫夫曼树）呢？

  1. 在森林中选出两棵根结点的权值最小的二叉树，小左大右放置。
  2. 合并两棵选出的二叉树，增加一个新的结点作为新二叉树的根，权值为左右孩子的权值之和。
  3. 从森林中选出一棵根结点权值最小的二叉树，与新生成的根结点小左大右放置。
  4. 合并选出的二叉树和新生成的树，增加一个新的结点作为新二叉树的根，权值为左右孩子的权值之和。
  5. 重复3、4。

赫夫曼编码

Huffman 编码是首个实用的压缩编码方案。在数据通信中，用二进制给每个字符进行编码时不得不面对的一个问题是如何使电文总长最短且不产生二义性。根据字符出现频率，利用赫夫曼编码可以构造出一种不等长的二进制，使编码后的电文长度最短，且保证不产生二义性。

定长编码：像 ASCII 编码，每一个字符编码要8位。

变长编码：单个编码的长度不一致，可以根据整体出现频率来调节。

前缀码：任一字符的编码都不是另一字符编码的前缀。

  1. build a priority queue 创建优先级队列，优先级取决于个字符在文本中出现的次数，次数越多，权重越大。
  2. build a Huffman tree 创建一棵赫夫曼树，（从优先级队列中取出两个最小的结点，生成一个新结点，再根据新结点的优先级，插入到队列的合适位置中。）
  3. build a Huffman table 创建一个赫夫曼表，根据遍历树的结果，左子树都用0表示，右子树都用1表示，为每个叶子结点生成一个表格。
  4. encode ，再次遍历需要压缩的文本或文件，找到对应的表和字符，将其 code 打印，写入另一个文件。
  5. decode ，根据字符的code 遍历赫夫曼树即可。
