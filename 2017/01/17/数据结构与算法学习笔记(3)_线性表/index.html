线性表（List）

线性表：零个或多个数据元素的有限序列。（点名册、星座列表）

线性表的个数n(n>=0)定义为线性表的长度，当n=0时，称为空表。

有序，前驱元素，后继元素， 个数>=0，数据类型相同。

在较复杂的线性表中，一个数据元素可以由若干个数据项组成。

线性表的抽象数据类型

  • 创建和初始化( createList(), initList() )
  • 判断是否为空表( listEmpty() )
  • 重置为空表( clearList() )
  • 根据位序得到数据元素( getElem() )
  • 查找是否存在( locateElem() )
  • 线性表长度( listLength() )
  • 插入和删除( listInsert(), listDelete() )

线性表的物理存储结构：

  • 顺序存储结构：用一段地址连续的存储单元依次存储线性表的数据元素。（抢占妹子旁边位置，依次占座）

    • 因为有票，所以即使没人也会空着
    • 结构封装：起始位置、最大存储容量（数组长度）、当前长度（当前数据元素个数）
    • 地址计算： 因为存储位置关系： Loc(ai+1) = Loc(ai)+c => Loc(ai) = Loc(a1)+(i-1)*c，可以随时计算出线性表中任意位置的地址。且都是相同的时间，所以存储时间性能为 O(1)，通常称为随机存储结构。
    • 整表创建：一维数组初始化
    • 插入操作的算法
      • 每次插入都需要移动n-i个位置，每次都是 O(n)。
    • 删除操作的算法
      • 最好的情况： O(1)
      • 最坏的情况： O(n)
      • 平均情况：O(n)
    • 存、读O(1) ，删、插O(n)。 适合元素个数较稳定、不经常插、删，更多是存取

    优：

    • 无须为表示表中元素间的逻辑关系而增加额外的存储空间
    • 可快速存取表中任意位置的元素

    缺：

    • 插、删需移动大量元素 （因为是顺序存储结构）
    • 长度变化大时，难确定存储空间容量
    • 易造成存储空间的”碎片“（中间会有小块存储空间）

  • 链式存储结构：用一组任意的存储单元存储线性表的数据元素，这组存储单元可以存在内存中未被占用的任意位置。但除了数据还需要存储后继元素的存储地址（指针）

    • 数据域：存储数据信息

    • 指针域：存储直接后继位置（指针/链）

    • 线性表的链式存储结构：n个节点链接成一个链表

    • 头指针（必要，常用其冠以链表名）：第一个节点的存储位置，若链表有头结点，则指向头结点。

    • 头节点（不必要），为了操作的统一和方便而设计的（统一第一个元素节点前的插入节点和删除第一节点），位于第一个元素的节点之前。

      • 数据域不存储任何信息，或存链表长度

    • 最后一个节点 an+null

    • 空指针：头指针+头结点+null 或头指针+null

    • 单链表的整表创建：动态生成

      • 头插法：从一个空表开始，生成新节点，读取数据存放到新节点的数据域中，然后将新节点插入到当前链表的表头上，直到结束为止。（始终让新节点插在第一的位置）但生成的链表中结点的次序和输入的顺序相反

    • 单链表的读取：工作指针后移 O(n)（使用while循环）

    • 单链表的插入、删除 O(n)

      • i位后连续插入10个元素，只需在第一次找到第i个位置的指针，此时为 O(n) ，之后只需赋值和移动指针，都是 O(n) 。因此，对插入或删除数据越频繁的操作，单链表的效率优势越明显。

静态链表

静态链表：用数组描述的链表，游标实现法。

  • 对静态链表进行初始化相当于初始化数组。
  • 对数组的第一个和最后一个元素做特殊处理，他们的 data 不存放数据
  • 通常把未使用的数组元素成为备用链表
  • 数组的第一个元素，即下标为 0 的那个元素的 cur 就存放备用链表的的第一个节点的下标
  • 数组的最后一个元素，即下标为 MAXSIZE-1 的 cur 则存放第一个有数值的元素的下标，相当于单链表中的头结点作用。
  • 静态链表的插入（在数组中，实现不移动元素，却插入了数据）

静态链表优点：

  • 在插入和删除操作时，只需要修改游标，不需要移动元素，从而改进了在顺序存储结构中的插入和删除操作需要移动大量元素的缺点。

静态链表缺点：

  • 没有解决连续存储分配（数组）带来的表长难以确定的问题。
  • 失去了顺序存储结构随机存取的特性。

总的来说，静态链表是为了给没有指针的编程语言设计的一种实现单链表功能的方法。（理解其思想）

环

环：链表的尾节点指向了链表中的某个节点。

判断方法：

  • p、q两个指针从头开始走，p每次前进一步，而q每次都从头开始走到p的位置（即两者值相等时），如果两者的步数不一样，则存在环。
  • 快慢指针：p、q两个指针从头开始走，p每次前进一步，q每次前进两步，若在某时 p==q ，则存在环。
