基础知识

web和移动应用：

  1. 用户在浏览器（客户端）里输入或点击一个链接
  2. 浏览器向服务器发送HTTP请求
  3. 服务器处理请求，如果查询字符串或者请求体里含有参数，服务器也会把这些参数信息考虑进去。
  4. 服务器更新、获取或者转换数据库里的数据
  5. 服务器以HTML、JSON、或者其他格式返回一个HTTP响应
  6. 浏览器接收HTTP响应
  7. 浏览器以HTML或者其他格式（如JPEG、XML或者JSON）把HTPP响应呈现给用户。

移动应用用远胜应用取代了浏览器，带宽带来的数据传输限制、更小的屏幕、更高效地使用本地存储。

原始string 和 Sting 对象的区别

每次读取一个基本类型值时，后台会创建一个对应的基本包装类型的对象。

使用 new 操作符创建的引用类型的实例，在执行流离开当前作用域之前都一直保存在内存中。而自动创建的基本包装类型对象，则只存在于一行代码的执行瞬间，然后立即被销毁。所以无法在运行时为基本类型值添加属性或方法。

敏捷开发

  1. Scrum 方法

    由一个个短周期组成，每个周期叫 sprint 。一个 sprint 持续1-2周。每天的会议会把新任务分配给团队成员。新任务不能添加到正在进行的 sprint 里，它们只能在子sprint 计划会议上添加。5-15分钟的会议中，每个团队成员要回答3个问题：自己昨天做了什么；今天准备做什么；是否需要从其他团队成员那里得到些什么。

    优势：在没法提前规划，或者需要收集用户反馈作为主要决策因素时，非常高效。

  2. 测试驱动开发（TDD）

    1. 使用assertion（true or false）为新功能、新任务或者增强写下失败的自动测试用例；
    2. 写出能够成功通过测试的代码
    3. 如果需要，重构代码，添加功能，同时保证测试用例通过
    4. 重复上面的步骤直到所有任务都完成

    优势：更少的错误和缺陷；更有效的代码；确保代码可以正常工作，并且不会影响已有功能。

  3. 持续部署（CD）和集成

    是一系列技术的组合，可以快速把开发好的新功能、错误修复和增强功能呈现给用户。CD 包含自动测试和自动部署。

    优势：减少反馈时间和手工劳动

  4. 结对编程

    两个开发者在同一个环境里一起工作。其中一个为”驾驶员“，另一个为”观察员“。驾驶员负责写代码，观察员围观并提供建议。一段时间后两者互换角色。驾驶员的角色更多关注当下的任务，观察员需要有大局观，发现错误并改进算法。

    优势：产出更简洁、高效的代码，同时降低引发错误和缺陷的几率。两人一起工作时彼此可以交流和传递知识。

Node.js

事件驱动异步I/O。可用于创建可伸缩且高性能 Web 服务器。

NoSQL和MongoDB

用于大数据的高性能非关系型数据库。

优势：

  • 可伸缩性：由于其分布性特性，可以在多个服务器和数据中心放置冗余数据
  • 高性能：在存储和检索方面非常高效，部分原因是数据库中元素和集合之间关系上的弱处理；
  • 灵活性：键-值对存储非常适合原型开发，因为它使用户不需要关心表关系，不需要修复数据模型，不需要关注复杂的数据迁移。

云计算

  • 基础设施即服务（IaaS），如 Rackspace、Amazon Web Services
  • 平台即服务（PaaS），如 Heroku、windows Azure
  • 后端即服务（Baas），如Parse.com、Firbase
  • 软件即服务（Saas），如 Google Apps、Saleforce.com

云应用平台功能：

  • 可伸缩性，如在几分钟内产生新的实例；
  • 部署简单，向Heroic 部署，只需要 $ git push
  • 现收现付制，用户根据需要添加和删除内存及磁盘空间
  • 可扩展，方便安装和设置数据库、应用服务器、程序包等
  • 安全和技术支持

PaaS 和 BaaS 非常适合构建原型，一般用来创建 MVP（minimal viable produce 最小化的可行产品），供那些尚处于创业初始阶段的团队使用。
