认识

$(document).ready(function(){}) vs window.onload

  • 执行时机

    前者在DOM完全就绪时就可以被调用，后者还需等元素所有关联文件加载完成

  • 编写个数

    前者事件能够保存对多个函数的引用

  • 简写

    前者可简写为 $().ready(function(){}) $(function(){})

与DOM对象的转化

  • $jq[index] $jq.get(index)
  • $(dom)

$cr.is(:checked) 与 cr.checked

解决冲突：

  • jQuery.noConflict() 后使用jQuery
  • jQuery.noConflict() 后使用jQuery(function($){$(‘p’…)})
  • jQuery.noConflict() 后使用(function($){$(‘p’…)})(jQuery)

优势

  • 写法简洁

  • 支持CSS1~CSS3的选择器

  • 完善的处理机制

基本选择器：id 类 标签 通配符 集合

层次选择器：后代 子元素 相邻元素 同辈元素

DOM

结点操作

查找节点

  1. 查找元素节点 $(‘ul li:eq(1)’)
  2. 查找属性节点 $p.attr(‘title’)

创建节点

  1. 创建元素节点 $(html).append($jq)
  2. 创建文本节点 $('<p>ccc</p>')
  3. 创建属性节点 $('<p title='...'>ccc</p>')

插入结点

  • A.append(B) B.appendTo(A) (也可以用来移动元素)
  • A.prepend(B) B.prependTo(A)
  • A.after(B) B.insertAfter()
  • A.before(B) B.insertBefore()

删除结点

  • remove() 删除某一结点包含的所有结点以及其自身，以及所有事件和附加数据

    var $li = $(‘ul li:eq(1)’).remove(); .remove(‘li[title!=XX]’)

  • detach() 删除某一结点包含的所有结点以及其自身，但绑定的事件和数据会保留下来

    var $li = $(‘ul li:eq(1)’).detach();

  • empty()清空结点中的所有内容但不包括其自身

复制结点

  • $jq.clone([true]).appendTo(‘body’)，添加true时，会同时复制元素中所绑定的事件

替换结点

  • A.replaceWith(HTMl|DOM)
  • $(HTMl|DOM).replaceAll(A)

包裹结点

  • $(‘strong’).wrap(‘‘)，将所有元素单独包裹
  • $(‘strong’).wrapAll(‘‘)，将所有匹配元素用一个元素包裹（如果被包裹的多个元素之间有其他元素，其他元素会被放到包裹元素之后）
  • $(‘strong’).wrapInner(‘‘)，将每一个匹配的元素的之内容（包括文本结点）用其他结构化的标记包裹起来。

遍历节点

  • children()，取得元素的子元素集合
  • next()，取得元素后面紧邻的同辈元素
  • prev() ，取得元素前面紧邻的同辈元素
  • siblings()，取得元素前后所有同辈元素
  • closest(‘li’)，从当前元素本身开始（包括当前元素）向上逐级查找，返回最先匹配的祖先元素li
  • .parent()，获得集合中每个匹配元素的父级元素
  • .parents(‘ul’)，获得集合中每个匹配元素的祖先元素，最后返回多个父节点
  • .find(‘li’)，取得元素内的所有li元素
  • .filter(‘li’)，取得集合中为li元素的元素

  • $(‘p’).removeAttr(‘title’)

判断是否含有样式

事件

加载DOM

  • $(function(){…})

事件绑定

  • $jq.on(type [,data] , fn)

    • type 可以为 click、dbclick、focus….也可以为自定义事件，可以是多个事件”click dbclick“
    • 可为事件添加命名空间 “click.namespace”

  • data将作为event.data属性值传递给事件对象

  • click、mouseover、mouseout等事件可简写 $jq.click([data,] fn)

  • $jq.once(type [,data] , fn) 触发一次后删除

合成事件

  • $jq.hover(enterFunc, leaveFunc)
  • $jq.toggle(fn1, fn2, …, fnN) 模拟鼠标连续单击事件

事件冒泡和默认行为

  • 停止事件冒泡 event.stopPropagation()
  • 阻止默认行为 event.preventDefault()
  • 同时阻止事件冒泡和默认行为 return false;

事件对象属性

  • event.type 事件类型
  • event.target 触发事件的元素
  • event.relatedTarget 用于访问mouseover和mouseout的相关元素
  • event.pageX 和 event.pageY 鼠标在页面中的位置
  • event.witch() mousedown、mouseup事件，1=左键，2=中键盘，3=右键
  • event.witch() keydown、keyup事件，获取按键
  • event.metaKey，获取ctrl按键

移除事件

  • $jq.off([type], [data])
    • 没有参数，删除所有绑定事件
    • 只有事件类型参数，只删除该类型事件
    • 有处理函数作为第2参数，只删除该特定的事件处理函数
    • “.namespace”，可删除该命名空间下的所有事件

事件模拟

  • $jq.tigger(eventType [,data])，可以是自定义事件
    • “click!”只触发click，而不触发click.namespace
    • “click”两者都触发
  • $jq.tiggerHandler(eventType [,data])，值触发绑定事件，取消浏览器默认操作
  • 简写 $jq.click()

动画

出现和消失

  • show([millseconds | 'fast' | 'normal' | 'slow' ] [，callback])
  • hide()
  • fadeIn()
  • fadeOut()
  • slideUp()
  • slideDown()

自定义动画方法

  • animate(params [, speed] [, callback])
    • $(this).animate({left: “200px”}, 3000)，向右移动200px
    • $(this).animate({left: “+=200px”}, 3000)，当前位置累加200px
    • $(this).animate({left: “200px”，height:”100px”}, 3000)，同时执行
    • $(this).animate({left: “200px”}, 3000).animate({height:”100px”},200)，顺序执行
    • $(this).animate({left: “200px”}, 3000，function(){$(this).css(…)})，最后执行回调

停止动画

  • stop([clearQueue], [gotoEnd])
    • 是否清空未执行完的动画队列，是否直接跳到末状态
    • 没有参数，这结束当前正在进行的动画，并立即执行队列中下一个动画

判断是否处于动画状态

  • $jq.is(‘:animated’)

延迟动画

  • $jq.animate(…).delay(millseconds)

交互动画

  • toggle(speed [, callback])
  • slideToggle(speed [, easing] [, callback])
  • fadeTo(speed, opacity [, callback])
  • fadeToggle(speed [, easing] [, callback])

编写

插件种类

  1. 封装对象方法的插件

    $jq.xx，如$jq.parent()

  2. 封装全局函数的插件

    $.xx，如$.ajax()

  3. 选择器插件

    如color(red)选择所有红色字的元素

基本要点

  • 命名为 jquery.插件名.js
  • 所有对象方法应附加到jQuery.fn对象上，所有全局函数应附加到jQuery对象本身上
  • 插件内部this指向当前通过选择器获取的jQuery对象
  • 可通过this.each遍历所有元素
  • 所有的方法或插件，都应当以分号结尾，甚至可在插件头部先加上一个分号
  • 插件应返回一个jQuery对象，以保证插件的可链式操作。除非插件需返回的是一些需要获取的量
  • 避免在插件内部使用$作为jQuery对象的别名，而应该用完整的jQuery表示，以避免冲突。也可利用闭包，是内部继续使用$作为别名。

闭包

  • 允许使用内部函数（即函数定义和函数表达式位于另一个函数的函数体内），而且，这些内部函数可以访问它们所在的外部函数中声明的所有局部变量、参数和声明的其他内部函数，当其中一个这样的内部函数在包含它们的外部函数之外被调用时，就会形成闭包。也就是内部函数会在外部函数返回后被执行，而当这个内部函数执行时，它仍然必须访问其外部函数的局部变量、参数以及其他内部函数。

插件机制

  • jQuery.fn.extend()用于扩展第1种类型的插件
  • jQuery.extend()用于扩展第2、3种类型的插件

jQuery mobile

用来填补 jQuery 在移动设备应用上的缺憾，它基于 jQuery 框架并使用了 HTML5 和 CSS3 这些新技术，除了能提供很多基础的移动页面元素开发功能外，框架自身还提供了很多可供扩展的 API，以便开发人员在移动应用上使用。

主要特性

  • 基于 jQuery 构建
  • 兼容大部分手机平台
  • 轻量级的库
  • 模块化结构
  • HTML5 标记驱动的配置
  • 渐进增强原则
  • 强大的 Ajax 的导航系统
  • 易用性
  • 支持触摸和鼠标事件
  • 统一的UI组件
  • 强大的主题框架
