类型转换大解析

发表于 2017-03-22   |   分类于 JavaScript   |     |   阅读次数

JavaScript 作为一门弱类型的语言，在调用方法、作判断和作比较的时候会在不经意间改变值的类型，如果在开发过程中不注意的话，一不小心就会掉到坑里。另外，前端笔试中经常会出现类型转换的题目，因此，你懂的～

转为 Boolean 类型

在JS中，所有类型的值（或者说变量）都可以转换为布尔值。在 if() 语句作条件判断，以及与或非（&&、||、!）等布尔操作时，都存在隐式类型转换。

当然，我们也可以手动显式地将某个变量转换为布尔值，一般有两种方法：

!! xx

Boolean(xx)

那重点来了，究竟哪些变量会被转换为 true，哪些变量会被转换为 false 呢？

  • 会被转为false的变量：undefined、null、false、0、NaN、空字符串。
  • 会被转为true的变量：除了会被转为 false 的值(＾－＾)V，如：true、除 0 和 NaN 以外的任何数字、任何对象、任何非空字符串。

好吧，如果上面的总结不利于记忆，我下面再附上《高级程序设计》中的表格：

数据类型 转换为 true 的值 转换为 false 的值
Undefined / undefined
Boolean true false
Number 任何非零数字值（包括无穷大） 0和NaN
String 任何非空字符串 “”
Object 任何对象 null

转为 Number 类型

我们可能经常需要显式地把一些字符串转换为数字，或者将数字在几种进制之间转换；我们还可能经常困惑于到底何时结果为 NaN 。下面就来总结一下吧~（记住，转换为数字后，最终都会显示为十进制。）

先来看看 Number(xx) 方法对于变量的转换。

Number(xx)

数据类型 输入 输出
null null 0
undefined undefined NaN
boolean ture/false 1/0
number 任意数字（123，NaN） 输入（123，NaN）
string 只包含数字（011） 忽略前导零，转为十进制（11）
string 包含有效浮点格式（01.1） 忽略前导零，转为浮点数值（1.1）
string 包含有效16进制（0xf） 转为十进制（15）
string 空字符串 0
string 包含其他格式的字符（011b） NaN
object 任意对象 调用对象的 valueOf() 方法，如果转换的结果是 NaN,则调用toSting()方法。

由于 Number() 方法在转换字符串时比较复杂而且不够合理，因此通常用 parseInt() 和 parseFloat() 方法转换。

parseInt(xx, x)

会忽略字符串前面的空格，直到找到一个非空格字符。如果第一个字符不是数字字符或者负号，parseInt() 就会返回NaN

数据类型 输入 输出
null null NaN
undefined undefined NaN
boolean ture/false 1/0
number 任意数字（123，010） 输入（123，8）
string 只包含数字（011bc） 忽略前导零，忽略非数字及之后的字符，转为十进制（11）
string 包含有效浮点格式（01.1） 忽略前导零，忽略非数字及之后的字符，转为十进制（1）
string 包含有效16进制（0xf） 转为十进制（15）
string 空字符串 NaN
string 第一个非空格字符不是数字字符或者负号 NaN
object 任意对象 调用对象的 valueOf() 方法，如果转换的结果是 NaN,则调用toSting()方法。

可以为这个函数提供第二个参数：转换时使用的基数。声明待转换的字符串是多少进制。

parseFloat(xx)

与 parseInt() 类似，parseFloat() 也是从第一个非空格字符开始解析，一直解析到字符串末尾，或者解析遇见一个无效的浮点数字字符为止。该方法可识别所有浮点数值格式，以及十进制整数格式。但十六进制格式的字符串始终会被转换为0。

如果字符串包含的是一个可解析为整数的数（没有小数点，或者小数点后都为 0 ），该方法会返回整数。

++ 、– 、+ xx 、 -xx

以上操作在应用于非 number 类型的值时，先使用 Number() 将其转换为数值（结果为NaN 或者 number类型的数字），再执行加减一的操作。

当 xx + number 或 xx- number 时，会先使用 Number() 将 xx 转为数值。（null+1 结果为 1 , undefined+1 结果为 NaN）。

isNaN(xx)

  • 任何数值除以非数值都会返回 NaN
  • 任何涉及 NaN 的运算操作都会返回 NaN
  • Infinity * 0、Infinity / Infinity、0 / 0、Infinity % 有穷、有限大 % 0、Infinity % Infinity、Infinity + (-Infinity)、undefined+number

在应用于非 number 类型的值时，先使用 Number(xx) 将其转换为数值（结果为NaN 或者 number类型的数字）

转为 String 类型

xx.toString(x)

该方法可以指定输出的进制（默认为十进制）

但是 null 和undefined 没有该方法，因此，在不确定转换值类型的情况下，可以使用 String() 。

String(xx)

如果值有 toString() 方法，则调用该方法并返回相应的结果。

如果值是 null，则返回 'null'

如果值是 undefined，则返回 'undefined'

xx + ‘string’

会先使用 String(xx) 将其转换为字符串（null+'1' 结果为 'null1'，undefined+'1' 结果为 'undefined1'）。

关系操作符中存在的类型转换

  1. 如果两个操作数都是数值，则执行数值比较。
  2. 如果两个操作数都是字符串，则比较两个字符串对应的字符编码值。
  3. 如果一个操作数是数值，则将另一个操作数转换为数值，然后执行数值比较。
  4. 如果一个操作数是对象，则调用该对象 valueOf() 方法，用得到的结果按照前面的规则进行比较。
  5. 如果一个操作数是布尔值，则先将其转化为数值，然后进行比较。
  6. 任何操作数与 NaN 进行关系比较其结果都是 false。('a' < 3、NaN < 3、NaN >= 3)

相等操作符中存在的类型转换

== 、!= 操作符都会先转换操作数（强制转型），然后再比较它们的相等性

  • 如果有一个操作数是布尔值，则在比较相等性之前先将其转换为数值（0或1）。
  • 如果一个操作数是字符串，另一个操作数，是数值则在比较相等性之前，先将字符串转换为数值。
  • 如果一个操作数是对象，另一个不是，则调用对象的valueOf()方法，得到基本类型值，再进行上述比较
  • null == undefined
  • 如果有一个数值是NaN，则相等操作符返回false，不相等操作符返回true。
  • 如果两个操作符都是对象，则比较它们是不是同一个对象。

=== 和 !== 操作符不会进行类型转换，直接比较相等性。
